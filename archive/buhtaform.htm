с формой [Стандартная. Декларация по НДС 2015. Экспорт в XML].

 

Форма [Стандартная. Декларация по НДС 2015. Экспорт в XML] добавлена в версии 2.163, применять следует за период  начиная с 1 квартала 2015 года.

Форма [Стандартная. Декларация по НДС 2015. Экспорт в XML] состоит из нескольких закладок, количество которых определяется настройкой параметров на главной закладке «Выгрузка».

Выгружаемые сведения разбиты на следующие группы, каждой из которой соответствует свой параметр, от значения которого зависит, осуществляется ли выгрузка этой информации или нет:

Разделы 1-7 – сведения выгружаются из архивного отчета, составленного по шаблону [Стандартный. Налоговая декларация по НДС 2015]. Если актуального отчета в архиве нет, его следует составить по кнопке <Новый отчет>. Если нужный отчет уже есть в архиве, его следует указать в поле «Архивный отчет» выбором из предоставляемого списка (после составления нового отчета ссылка на него проставляется в это поле автоматически). При этом в файл XML, соответствующий разделам 1-7, выгружаются названия файлов по разделам 8-12, поэтому надо либо одновременно создавать и их тоже, либо указывать названия ранее созданных файлов в соответствующих полях, которые становятся доступны для редактирования при выставленном параметре «Из файла».

Раздел  8 – сведения выгружаются из книги покупок по тем разделам (файл выгрузки с префиксом NO_NDS.8) и из дополнительных листов книги покупок (файл выгрузки с префиксом NO_NDS.81). Если установлены параметры для этих выгрузок, на форме появляется дополнительная закладка Книга покупок, на которой можно просмотреть данные, подлежащие выгрузке.

Раздел  9 – сведения выгружаются из книги продаж по тем разделам (файл выгрузки с префиксом NO_NDS.9) и из дополнительных листов книги продаж (файл выгрузки с префиксом NO_NDS.91). Если установлены параметры для этих выгрузок, на форме появляется дополнительная закладка Книга продаж, на которой можно просмотреть данные, подлежащие выгрузке.

Раздел 10 – сведения выгружаются из журнала учета выставленных счетов-фактур. Если установлены параметры для этой выгрузки, на форме появляется дополнительная закладка «Журнал выставл. с/ф», на которой можно просмотреть данные, подлежащие выгрузке. Файл этой выгрузки имеет префикс  NO_NDS.10.

Раздел 11 – сведения выгружаются из журнала учета полученных счетов-фактур. Если установлены параметры для этой выгрузки, на форме появляется дополнительная закладка «Журнал получ. с/ф», на которой можно просмотреть данные, подлежащие выгрузке. Файл этой выгрузки имеет префикс NO_ NDS.11.

Внимание!

- Сведения из Книг покупок/продаж и Журналов учета выгружаются только по тем разделам, у которых в настройке выставлен параметр «Включать в декларацию по НДС».

- Дополнительные листы Книг продаж и покупок надо выгружать только при выгрузке Уточненной Декларации – должен быть выставлен соответствующий параметр на форме, при этом надо также указать Номер корректировки  и Признак актуальности ранее представленных сведений из Книг и Журналов. Если сведения актуальны, на форме следует выставить параметр «АКТУАЛЬНЫ ранее представленные сведения из Книг и Журналов», в этом случае сведения из Книг и Журналов не будут попадать в соответствующие файлы.

 

В версии 2.171 был добавлен режим создания ответа на Требование ИМНС о представлении пояснений по Налоговой декларации по НДС.

При нажатии на кнопку <Добавить Ответ на Требование> в таблицу [_ДеклНДС_Пояснение] добавляются записи с типом Подтверждение с указанными реквизитами Требования (номер и дата) по всем отмеченным записям списка текущей закладки (Книги покупок-продаж или  Журнала СФ). Тип Подтверждение означает, что Сведения, в отношении которых направлено Требование, соответствуют первичным учетным документам и не допущены ошибки заполнения декларации. В случае, если в Сведениях, в отношении которых направлено Требование, выявлены ошибки заполнения декларации, при этом первичные учетные документы не содержат указанных ошибок, надо присвоить этому сведению тип Расхождение и для каждого такого сведения добавить строку с типом Пояснение, в которой заполняются только корректируемые значения исходной записи.

Перевод сведения из Подтверждения в Расхождение и добавление соответствующего сведения Пояснение делается автоматически по всем отмеченным сведениям при нажатии на кнопку <Перевести в Расхождение> на закладке Ответ на Требование о пояснениях, на которую выведен список записей новой таблицы [_ДеклНДС_Пояснение]. Этот список обновляется по нажатию:

1) кнопки  <Выборка> - выводятся все записи,

2) кнопки <Показать ответ на Требование> - выводятся записи с указанным номером.

Выгрузка пояснений в файл производится по нажатию кнопки <Выгрузить> на закладке Ответ на Требование о пояснениях.

Само Требование загружается на этой же закладке по нажатию кнопки <Загрузить из XML-файла с Требованием>. При поиске на имя файла с Требованием наложено условие "1165050*.xml".

Для Раздела 2 следует настроить новый запрос [Стандартный. Налоговая декларация по НДС Раздел 2]. Особенности его настройки:

- значение для строки 050 (код ОКАТО) берется из текущего юр.лица

- для строк 040 (КБК) и 070 (Код операции) в запросе предусмотрена заглушка в виде SQL_Строк – настраивается пользователем.

2.176

В режим составления «Налоговой декларации по транспортному налогу» добавлен алгоритм расчета повышающего коэффициента в зависимости от средней стоимости  и количества лет, прошедших с года выпуска автомобиля. Повышающий коэффициент с 2015 г. применяется как при подаче декларации, так и при расчете авансовых платежей.

Поскольку порядок расчета средней стоимости легковых автомобилей определяет федеральный орган, регулирующий сферу торговли - Минпромторг(п. 2 ст. 362 НК РФ), то перед подачей декларации или расчетом авансового платежа пользователь должен свериться с Перечнем легковых автомобилей стоимостью от 3 млн руб. и проставить нужное значение (в рублях) в поле [_Стоимость ТС для налога] карточки ОС/

Так в отношении автомобилей средней стоимостью от 3 до 5 млн руб. повышающий коэффициент равен 1.5, 1.3, 1.1, если количество лет  прошедших с года выпуска автомобиля соответственно не более одного (0 и 1), двух или трех (для цены ровно 3 млн.руб коэффициент равен 1);

если средняя стоимость автомобиля составляет от 5 до 10 млн руб. и с года выпуска прошло не более 5 лет, то коэффициент равен 2;

если с года выпуска транспортного средства стоимостью от 10 до 15 млн руб. прошло не более 10 лет, то коэффициент равен 3.

Этот же коэффициент 3 используется в случаях, если средняя стоимость превышает 15 млн руб. и автомобиль выпущен не более 20 лет назад.

В остальных случаях, в том числе при незаполненном поле [_Стоимость ТС для налога], коэффициент принимает значение равное 1.

Обращаем особое внимание на то, что если в карточке ОС год выпуска не указан, количество лет использования полагается равным нулю (т.е менее 1 года).

Сведения о состоянии расчетов за отгруженную продукцию, выполненные работы (услуги) - Форма № 1-РП (срочная)

Версия: 1.88

Описание:
В соответствии с приказом Росстата от 07.08.2008 N 184 "Об утверждении форм статистического наблюдения для организации статистического наблюдения за ценами и финансами на 2009 год" начиная с отчета за 2008 г., юридические лица, независимо от формы собственности, которые осуществляют любые виды экономической деятельности (кроме субъектов малого предпринимательства, банков, страховых и бюджетных организаций), должны подавать в территориальный орган по месту своего нахождения

отчет по форме N 1-РП (срочная), в которой содержатся сведения о состоянии расчетов за отгруженную продукцию, выполненные работы (услуги). Данная форма является одновременно месячной (подается не позднее 23-го числа месяца, следующего за отчетным, начиная с 2009 г.) и годовой (подается не позднее 30 января года, следующего за отчетным, начиная с отчета за 2008 г.). Причем годовой отчет должны подавать даже организации, перешедшие на УСН, и временно неработающие компании.

 

Отчет вызывается из пользовательской формы, на которой следует указать значения следующих параметров:

-          период, за который составляется отчет,

-          источник данных за аналогичный период предыдущего года, а именно каким образом в отчете будут формироваться данные за соответствующий период предыдущего года: по имеющимся проводкам или будут выбираться из ранее составленного отчета, хранящегося в архиве.

-          флаг "нарастающим итогом с начала года" – по умолчанию установлен.

Создание счетов-фактур на аванс

Версия: 2.136

Описание:

 

Форма предназначена для ежедневного отслеживания полученных авансов и их автоматического зачета. Основные условия применения формы - используются 3 бухгалтерских счета для учета Авансов, Оплаты, Реализации. На счете Аванса на 2-м субконто необходимо фиксировать платежные документы.

 

Пользователю следует указать значения параметров, запрашиваемые на форме, в том числе помимо перечисленных выше счетов -  вид документов для зачета аванса, оплаты и возвратов, и вид документа для счета-фактуры на аванс, а также вид спецификаций  для каждого случая.

 

Процедуры выполняются в указанном порядке за каждый день заданного периода:

- Зачет авансов закрывает сальдо Авансов на сальдо Реализации.

- Зачет оплаты текущего дня закрывает кредит Оплаты на сальдо Реализации.

- Зачет возвратов текущего дня  -  это зачет возврата денег с дебета Оплаты на сальдо Аванса (если оно имеется).

- СФ на аванс по возврату – создается счет-фактура, если образовалось кредитовое сальдо Реализации.

- СФ на аванс – создается счет-фактура по кредитовому сальдо Оплаты.

 

По окончанию работы процедур выводится список созданных документов. Пользовательская форма и запрос, которые используются при этом, входят в состав конфигурации BuhtaConfig.

Формы отчетности с настройкой показателей

В целях унификации подхода к составлению отчетности, основанной на выборках из сальдо и проводок, разработана новая система настроек для форм отчетности.

Во-первых, реализована принципиально новая схема настройки запросов для составления бухгалтерской и статистической отчетности. Запросы строятся в соответствии с настройками полей запроса, которые содержатся в специальной таблице [_Настройка форм отчетности]. Во-вторых, предусмотрено получение и отображение в отчетах данных из составленных ранее отчетов, которые сохраняются в специальной таблице [_Показатели архивных отчетов]. Данные в таблицу [_Показатели архивных отчетов] записываются при создании и изменении некоторых отчетов, составленных по шаблону типа MS Excel.

В рамках этого алгоритма разработаны следующие бухгалтерские отчеты и отчеты статистического наблюдения:

  • Приложение к бухгалтерскому балансу
  • Сведения о производстве и отгрузке товаров и услуг, форма № П-1. Сведения об объеме платных услуг населению по видам, приложение № 3 к форме № П-1
  • Сведения о финансовом состоянии организации, форма № П-3
  • Основные сведения о деятельности организации, форма № 1-предприятие
  • Сведения о затратах на производство и продажу продукции (товаров, работ, услуг), форма № 5-З
  • Сведения об основных показателях деятельности малого предприятия, форма № ПМ
  • Сведения о движении ценных бумаг и доходах по ним по институциональным секторам, форма № 1-ЦБ

 

Начиная с версии 1.91 добавлены новые варианты (формы) составления отчетов для уже имеющихся отчетов:

  • Бухгалтерский баланс
  • Отчет о прибылях и убытках
  • Отчет о движении денежных средств
  • Сведения о состоянии расчетов за отгруженную продукцию, выполненные работы (услуги) - Форма № 1-РП (срочная)

·   Изменен алгоритм составления Формы № П-1 с учетом наличия обособленных подразделений.

При наличии у юридического лица обособленных подразделений отчет должен быть составлен как по каждому обособленному подразделению, так и по юридическому лицу без этих обособленных подразделений. В связи с чем на экран формы для составления отчета [Стандартный. Сведения о производст. и отгрузке П-1] добавлен признак «Отдельные отчеты для обособл. подразделений (по ОКПО)». Если признак проставлен, то вместо одного отчета создается несколько. При этом группировка данных в один отчет производится по коду ОКПО обособленных подразделений (в карточке Подразделения должен быть проставлен признак «Обособленное подразделение» и заполнено поле ОКПО). Отдельные отчеты создаются для всех обособленных подразделений с одинаковым кодом ОКПО, который печатается на титульном листе и в примечании к названию архивного отчета. Данные по всем не обособленным подразделениям также попадают в отдельный отчет с кодом ОКПО текущего юр.лица.

Перед первичным составлением отчета при выставленном признаке «Отдельные отчеты для обособл. подразделений (по ОКПО)» следует через форму настройки запросов (вызывается нажатием командной кнопки с изображением треугольника справа от кнопки <Составить отчет П-1>) настроить следующие запросы:

- за текущий период - запрос [Стандартный. Форма П-1/1 по подразделениям],

- за предыдущий месяц- запрос [Стандартный. Форма П-1/2 по подразделениям],

- за месяц прошлого года, соответствующий текущему – запрос [Стандартный. Форма П-1/3 по подразделениям].

Для облегчения этой задачи в форму настройки запросов добавлена кнопка <Копировать из другого запроса>, по нажатию на которую  можно выбрать форму и запрос, показатели из которого надо скопировать в текущий запрос. В данном случае следует выбрать для формы значение П1, для запросов - значения:

- за текущий период  - запрос [Стандартный. Форма П-1/1],

- за предыдущий месяц – запрос [Стандартный. Форма П-1/2],

- за месяц прошлого года, соответствующий текущему – запрос [Стандартный. Форма П-1/3]

Затем требуется перенастроить скопированные показатели для работы с учетом обособленных подразделений. В частности должен быть перенастроен показатель [s2_21_B], который отвечает за код по ОКВЭД.

Также следует откорректировать запрос [Стандартный. Форма П-1/ОКЕИ по подразделениям], отвечающий за Раздел 5 отчета. Запрос, входящий в конфигурацию BuhtaForm, дается в качестве примера. Пользователю необходимо самостоятельно настроить значения полей: [s5_0_А], [s5_0_Б], [s5_0_В], [s5_0_Г], [s5_0_Д], [s5_0_1], [s5_0_2], [s5_0_3], [s5_0_4], [s5_0_5] и для параметра запроса Сальдо/Оборот выбрать из предлагаемого списка и установить значение [Стандартный.
