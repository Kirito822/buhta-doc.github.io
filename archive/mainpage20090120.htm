Книга покупок и книга продаж

 

[ + ] [14799], [15110]

Модернизированы режимы работы с Журналом учета полученных счетов-фактур и Журналом учета выставленных счетов-фактур соответственно для Книги покупок и Книги продаж (меню Данные - Книга покупок / Книга продаж, кнопка Журнал).  

 

• Для Журнала учета полученных с/ф и Журнала учета выставленных с/ф добавлен новый параметр - признак Не удалять при пересоставлении. Записи Журналов, у которых этот признак выставлен, не удаляются при последующем пересоставлении Книги.

 

•Разработаны HotKey-формы [Копирование строк книги покупок в журнал учета с/ф]  и [Копирование строк книги продаж в журнал учета с/ф], выполняющие копирование отмеченных строк из Книг в Журналы. Записи в Журналах  создаются с заполненным признаком Не удалять при пересоставлении. Формы вызываются соответственно из Книги покупок и Книги продаж - нажатие "горячих клавиш" Alt+F12 открывает окно Выбор обработчика со списком доступных форм, среди которых указанные.

 

Внимание!!! Если запись добавляется в Книгу вручную, то  она автоматически попадает в Журнал. Признак Не удалять при пересоставлении при этом имеет тоже значение, что и  признак Введено в ручном режиме у соответствующей записи книги.

 

•Для записей Журналов разработаны режимы Добавления, Изменения, Удаления и Копирования записей.

 

При добавлении записи:

- проставляется признак [Не удалять при пересоставлении]=1,

- дата устанавливается первым числом периода в карточке просмотра журнала,

- раздел заполняется первым встреченным в базе разделом из перечисленных в списке  журнала.

 

Режим изменения записи позволяет менять все поля Журнала, в том числе и поле Не удалять при пересоставлении.

 

При удалении удаляются только записи с признаком [Не удалять при пересоставлении]=1. При  множественном удалении выдается сообщение о наличии записей с не проставленным признаком [Не удалять при пересоставлении] с подтверждением на удаление записей с проставленным признаком.

 

При копировании копируется только текущая строка, у новой строки проставляется признак [Не удалять при пересоставлении]=1. Множественного копирования нет.

 

Отчеты

 

[ - ] [15047]

В Отчете о прибылях и убытках (меню Отчеты - Формы бухгалтерской отчетности - Отчет о прибылях и убытках) не учитывалась настройка Считать 1 января переходным периодом (устанавливается в режиме Настройка - Общая настройка).

При установке такой настройки обороты за 1 января включаются во входящее сальдо и исключаются из оборотов.

Для исправления ошибочной ситуации внесены изменения в запрос [Стандартный. #Отчет о прибылях и убытках 2006], который используется в этом отчете. Запрос входит в поставку стандартной конфигурации BuhtaForm.mdb, которая может быть импортирована в базу.

ВНИМАНИЕ!!! В том случае, если в клиентской базе уже имеется настроенный под собственные нужды запрос [Стандартный. #Отчет о прибылях и убытках 2006], не следует производить его импорт из файла BuhtaForm.mdb, а достаточно в область SQL-Before этого запроса добавить следующие строки:

 

DECLARE @Perehod BIT

SET @Perehod = ISNULL((SELECT Значение FROM [Настройка]  WHERE Секция = 'Общая настройка' AND Параметр = 'Переходный период'),0)

IF @Perehod = 1 AND DAY(@AlgoritmBegDate) = 1 AND MONTH(@AlgoritmBegDate) = 1 SET @AlgoritmBegDate = DATEADD(dd,1,@AlgoritmBegDate)

 

[ - ] [15063]

Обнаружена ошибка при составлении отчетов на SQLServer 2005 и старше, которая не проявлялась на SQLServer 2000. Проблема заключалась в том, что для подчиненных запросов в отчетах типа FastReport и Эл.таблица добавляется дополнительная сортировка, которая не учитывала то, что данные могли быть сгруппированы. Ошибка исправлена.

 

[ - ] [15091]

При составлении отчета на основе шаблона типа FastReport версии 4 и дальнейшем его редактировании (редактирование производилось до записи отчета в архив) отчет сохранялся в архив без выполненных в процессе редактирования изменений. Ошибка исправлена, теперь отчет сохраняется в архив со всеми внесенными изменениями.

 

Зарплата См. также Конфигурация ''Стандартная_Зарплата_...''

 

[ * ] [15046]

В режимах расчета заработной платы добавлена возможность вводить значения для налоговых вычетов по коду вычета. Для этого в таблицу Ставка по дате (вызывается из меню Зарплата - Настройка - Ставки на дату) добавлено поле Код, а в таблицу Ставка тип - новый тип ставки Налоговый вычет - изменения производятся при синхронизации базы эталоном текущей версии.

Теперь при расчете суммы вычета значения налоговых вычетов сначала ищутся по коду вычета и дате в таблице Ставка по дате и только в том случае, если таких значений нет, берутся значения, прописанные в функции, как  и ранее. Записи с типом ставки Налоговый вычет при синхронизации базы эталоном новой версии в таблицу Ставка по дате автоматически не добавляются - в случае необходимости заводятся вручную.

[ + ] [15024]

В справочник юридических лиц добавлено поле Не отображать при запуске. При запуске программы в списке юр. лиц не отображаются те строки, у которых значение этого поля равно 1 (в том случае, если значение 1 установлено у всех строк справочника - при запуске программы показывается полный список юр.лиц).  Значение поля Не отображать при запуске (по умолчанию =0) меняется через пользовательскую форму Установка параметров юр.лица, которая не входит в стандартную поставку.
