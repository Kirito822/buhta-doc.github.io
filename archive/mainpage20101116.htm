Справочники

 

[ + ] [17241]

Модернизирована форма копирования комплекта ТМЦ (вызывается из списка комплектующих - карточка редактирования ТМЦ, закладки Основной комплект, Дополнительные комплекты) - добавлен флаг В несколько ТМЦ (только в основной комплект), в зависимости от его значения доступен выбор либо одной ТМЦ, для которой будет производиться копирование, либо списка ТМЦ.

 

Автонумерация

 

[ - ] [17248]

При настройке автонумерации в окне редактирования вида справочников Договоры, Документы, Организация, Сотрудник, Подразделения, ОС, ТМЦ, Прочие в случае задания типа автонумерации Из вида не создавалась соответствующая процедура автонумерации, что приводило к ошибкам при ее вызове из тех режимов, где обращение происходило именно к ней, а не к процедуре того вида, на который она ссылалась.

Замечание: ошибка наблюдалась только в окне редактирования вида справочников, при запуске глобальной утилиты Создание хранимых процедур для автонумерации из режима Сервис- Целостность базы данных процедуры создавались в полном объеме.

 

[ * ] [17253]

Для таблиц имеющих разделение по видам и Юр.лицам (Документы, Договора, Кадры Приказ, Кадры Договор) настройка автонумерации в случае алгоритма Из вида работает следующим образом - в списке видов показываются  только те виды, у которых по каждому!!! юр.лицу имеется "своя" настройка автонумерации. В том случае, если таких видов нет, при попытке выбрать настройку Из вида теперь выдается  сообщение "Нет видов, которые могут служить основой для автонумерации "Из вида". В виде-основе должна быть настроена "Своя" автонумерация для ВСЕХ юр.лиц" и выбор настройки Из вида не сохраняется.

 

Зарплата

 

[ * ] [17271]

Исправлена последовательность предоставления налоговых вычетов при расчете подоходного налога - вычет 312 предоставляется после того, как полностью предоставлены вычеты 313 и 314. Изменения внесены в процедуру [з/п Расчет вычетов без доходов] и будут задействованы после синхронизации базы данных эталоном текущего релиза.

 

[ - ] [17281]

Если сотруднику в течение года была предоставлена инвалидная льгота, а потом снята (путем добавления соответствующих.записей в налоговые данные  - закладка Налоги в Личной карточке сотрудника) то в месяцах, следующих за месяцем снятия льготы, это снятие не учитывалось - при расчете заработной платы в расчетном листке для шифров проставлялся вид дохода ЕСН=13 Инвалид. Ошибка исправлена - сейчас в подобных ситуациях, после снятия инвалидной льготы ставится вид дохода ЕСН из карточки шифра.

Изменения внесены в процедуру [з/п удержание/Подоходный налог], которая будет обновлена при синхронизации базы эталоном текущей версии.
