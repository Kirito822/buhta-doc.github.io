Договоры и документы

 

[ + ] [20531]

В режим сохранения результатов работы ТСД добавлена возможность вызова пользовательской процедуры, которая осуществляет дополнительную обработку данных. Нужная процедура указывается в параметре Процедура переноса в договор / документ (карточка настройки вида  договора / документа, закладка Отображение, группа параметров Работа с ТСД)  выбором из предлагаемого списка процедур, который составляется  из процедур с одним целочисленным параметром, поэтому при написании необходимой пользовательской процедуры следует учитывать это условие.

 

[ * ] [20553]

В режимах экспорта  /импорта в файл /из файла конфигурации обеспечена передача значений полей [ТСД.Название режима], [ТСД.Процедура сохранения] из настроек вида документа и вида договора.

 

[ * ] [20562]

Введен запрет на работу в режиме ТСД, если в настройках прав для соответствующего вида договора (документа) в разделе Данные- Договоры /Виды договоров (Документы /Виды документов) – Спецификации стоит значение Чтение.

 

Отчеты

 

[ * ] [20508]

Для формы статистического наблюдения П-3 добавлен новый шаблон [Стандартный. Сведения о фин.сост. орг-ции П-3 2014], который вызывается  начиная с годового отчета за 2013 год.

Исправлена выгрузка отчета за периоды: 1-ый квартал, 1-ое полугодие, 9 месяцев, год - добавлена выгрузка данных из разделов  2, 3, 4.

Добавленные и измененные в рамках этой модернизации объекты:

- шаблон [Стандартный. Сведения о фин.сост. орг-ции П-3 2014],

- форма [Стандартный. Сведения о фин.сост. организации П-3],

- форма [Стандартная. Архивный отчет в электронном виде]

входят в конфигурацию BuhtaForm.

 

[ * ] [20509]

Исправлены формы [Стандартная. Архивный отчет в электронном виде] и [Стандартная. Архивный отчет в эл.виде (Стат.формы)] (конфигурация BuhtaForm) - добавлена выгрузка в электронный вид Формы № 5-З статистического наблюдения.

 

Безопасность, пароли, доступы

 

[ + ] [20341]

Разработан режим, показывающий реальные доступы пользователя, с учетом его собственных прав, а также прав групп и ролей, в которые он входит (меню Настройка - Пользователи / Пароли / Доступы, закладка Учетные записи, кнопка <Тек.доступ>).

 

Сервис

 

[ * ] [20537]

Модернизирован режим посылки сообщений. Теперь, если  сообщение присылается из другой базы, помимо имени пользователя указывается и имя базы, откуда пришло сообщение.  К тому же,  после нажатия на кнопку <Ответить> настройка адреса получателя ответа автоматически производится на отправителя начального сообщения.

Модернизация  требует обновления БД эталоном текущей версии.

 

Зарплата См. также раздел справки Конфигурация ''Стандартная_Зарплата_...''.

 

[ + ] [20541]

Добавлен новый тип Перечисления (Платежного поручения) - Перечисление по запросу, для него в карточке редактирования ЗП Перечисления (меню Зарплата - Настройка - Перечисление) открываются новые поля:

- поле Запрос:  на поле накладывается фильтр - все запросы по таблицам Докспец, Зарплата и Проводка. В качестве примера в файл конфигурации BuhtaConfig добавлены 3 таких запроса [Стандартный. Перечисление по запросу (Докспец)], [Стандартный. Перечисление по запросу (Зарплата)], [Стандартный. Перечисление по запросу (Проводка)]. Указанные запросы могут быть отредактированы - дополнены новыми полями, существующие поля типа SQL-строка, SQL-число изменению не  подлежат. В запросе должно быть обязательно поле Сумма, могут быть поля Счет, Тип субконто 1-5 и Субконто 1-5.

- поле Группировать Поручения по полю запроса: если Да - то открывается параметр для выбора группировки по полям запроса.

- для полей Плательщик, Р/С Плательщика, Получатель и Р/С Получателя добавлена возможность для выбора любого из численных полей указанного выше запроса, значение которого будет рассматриваться как ключ справочников Организация (для полей Плательщик, Получатель) и Расчетный счет (для полей Р/С Плательщика, Р/С Получателя). Напоминаем, значениями этих параметров карточки редактирования ЗП Перечисления заполняются соответствующие поля в создаваемых Платежных поручениях.

 

Чтобы распределять Платежные поручения по разным документам, следует указать хотя бы одно из полей: Группировка, Плательщик, Р/С Плательщика, Получатель или Р/С Получателя, по совокупности которых и будут создаваться отдельные платежные поручения.

 

Измененные и добавленные в рамках текущей модернизации объекты входят в конфигурацию BuhtaConfig.
