Книга покупок и книга продаж

 

[ * ] [20568]

В соответствии с Постановлением Правительства РФ от 30.07.2014 N 735 внесены следующие изменения в  Книги покупок и продаж, в журналы учета сч-фактур и в счета-фактуры:

 

Книга покупок

•На экранную форму добавлены поля:

- Номер документа оплаты

- Всего покупок, вкл.НДС (в валюте)

- Валюта

•В карточку редактирования  строки книги покупок добавлены поля:

- Посредник

- Номер документа оплаты

- Всего покупок с НДС в валюте

- Валюта

•В режим настройки книги покупок добавлена новая колонка раздела Всего покупок вкл. НДС в валюте

•Добавлены новые шаблоны: [Стандартный. Книга покупок 2014], [Дополнительные листы книги покупок 2014].

 

Книга продаж

•На экранную форму добавлены поля:

- Номер п/п

- Номер документа оплаты

- Всего продаж, вкл.НДС (в валюте)

- Валюта

•В карточку редактирования строки книги продаж добавлены поля:

- Посредник

- Номер документа оплаты

- Всего продаж с НДС в валюте

- Валюта

•Добавлен режим перенумерации строк книги продаж согласно  сортировки на экранной форме – вызывается по нажатию на кнопку <Перенумерация>.

•В режим настройки книги продаж добавлена новая колонка раздела Всего продаж вкл. НДС в валюте.

•Добавлены новые шаблоны: [Стандартный. Книга продаж 2014], [Дополнительные листы книги продаж 2014].

 

Журнал полученных счетов–фактур

•Добавлены следующие поля (для просмотра этих данных в экранной форме журнала следует отредактировать соответствующий запрос):

- Тип посредника

- Посредник

- Всего покупок с НДС в валюте

- Валюта

•В карточку учета журнала  добавлены поля:

- Тип посредника

- Посредник

- Всего покупок в валюте

- Валюта

•Добавлен новый шаблон: [Журнал полученных счетов-фактур 2014]

 

Журнал выставленных счетов–фактур

•Добавлены следующие поля (для просмотра  этих данных в экранной форме журнала следует отредактировать соответствующий запрос):

- Тип посредника

- Посредник

- Всего продаж с НДС в валюте

- Валюта

•В карточку учета журнала  добавлены поля:

- Тип посредника

- Посредник

- Всего продаж в валюте

- Валюта

•Добавлен новый шаблон: [Журнал выставленных счетов-фактур 2014].

 

Добавленные и измененные  в рамках модернизации объекты входят в состав стандартной конфигурации BuhtaConfig.

Модернизация также требует обновления БД эталоном текущей базы.

[ * ] [20557], [20596]

Внесены исправления в Акт сверки взаиморасчетов (меню Отчеты – Расчеты с дебиторами и кредиторами – Акт сверки взаиморасчетов):

- При указании  достаточно длинного списка счетов на форме задания параметров, в отчет не попадали данные по счетам, указанным в конце списка – ошибка исправлена.

Измененные в рамках этой заявки объекты:

1) запрос [Стандартный. Список договоров с выбором] со скриптом,

2) формы [Стандартная. Акт сверки взаиморасчетов 2010], [Стандартная. Список договоров с выбором],

3) шаблон [Стандартный. Акт сверки взаиморасчетов 2010]

входят в стандартную конфигурацию BuhtaConfig.

 

Формы

 

[ * ] [20340]

Исправлен мастер создания пользовательских форм (режим Настройка - Инструменты - Формы пользователя, кнопка <Добавить>). Под операционными системами Windows 7, Windows 8 объекты на формах, созданных с помощью мастера, выходили за края формы.

 

Сервис

 

[ * ] [20385]

Модернизирован режим Сервис – Переслать  в сообщении (вызывается из контекстного меню списка).  Теперь гиперссылку на объект можно открыть непосредственно в окне присланного сообщения (ранее эта ссылка открывалась только из списка сообщений в режиме Сервис – Сообщения). Гиперссылкой в данном контексте считается строка, заключенная между символами '<' и '>' и содержащая символ '='. Ссылки в сообщении, присланном  из другой базы, цветом не выделяются и на клик мыши не реагируют.

 

Настройка

 

[ * ] [20543]

•Для работы с таблицами, имеющими разделение по видам, изменены алгоритмы определения запроса для просмотра по настройкам карточек редактирования таблиц по виду и в режиме Настройка – Инструменты – Подключение форм.

Теперь ,если на данные в списке наложен фильтр по виду, то сначала в качестве запроса для просмотра ищется значение соответствующего параметра в настройке карточки редактирования по этому виду, если его нет, берется  запрос, указанный в настройке Подключение форм по этой таблице и этому виду, если такой записи нет, то  ищется настройка по этой таблице с наименьшим видом (в том числе и 0), если такой нет - берется запрос с именем, совпадающим с именем таблицы.

Если данные в таблице представлены в полном объеме (выбран фильтр по видам <Все>), то в качестве запроса для просмотра берется запрос из настройки Подключение форм с видом равным 0, если такого нет, то берется запрос с именем, совпадающим с именем таблицы.

•Исправлена процедура вызова карточки редактирования строк Сводного отчета – неправильно определялся запрос, на основе которого был построен отчет.

•Исправлена ошибка при выборе запроса для списка спецификаций в карточке договора при ее вызове из Прайс-листа.

Информация для разработчиков

 

[ + ] [19421]

В VB скрипты карточки редактирования для объекта Form добавлено событие AfterGridLoadData.

 

[ + ] [20611]

В окно Кто работает в сети (меню Сервис - Кто работает в сети) добавлена поддержка VB скрипта. Доступны события: OnCreate, OnShow, OnLoadData, OnClose. В скрипте доступны кнопки, подписи по именам объектов.

 

Пример написания скрипта:

Sub FormOnLoadData ''

  if AllBases.Value then

    AllLicenseLabel.Caption = "***"

    AllLicenseUsedLabel.Caption = "* скрыто *"

    AllMobileLicenseLabel.Caption = "* скрыто *"

    AllMobileLicenseUsedLabel.Caption = "* скрыто *"

    AllMobileBaseLicenseLabel.Caption = "* скрыто *"

    AllMobileBaseLicenseUsedLabel.Caption = "* скрыто *"

    AllLiteLicenseLabel.Caption = "* скрыто *"

    AllLiteLicenseUsedLabel.Caption = "* скрыто *"

  end if

End Sub
