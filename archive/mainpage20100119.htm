Договоры и документы

 

[ * ] [16542]

В карточки копирования документов/договоров добавлен флаг Копирование без блокировки (режим копирования вызывается из контекстного меню или по нажатию клавиш Ctrl+C в списках документов/договоров).

 

При выставленном флаге Копирование без блокировки в процессе копирования не происходит блокировки работы других пользователей с таблицами Документ/Договор, что существенно при пакетном копировании большого числа документов/договоров.

Изменения в процессе копирования при выставленном флаге Копирование без блокировки:

1. Процесс копирования несколько замедляется.

2. Если в процессе копирования документа/договора по отмеченным записям произойдет ошибка, например, отмечено  4 документа/договора, и при копировании 3-его возникла ошибка, скопированными окажутся 2 документа/договора.

3. Недоступен режим копирования в один документ/договор.

Настройка

 

[ + ] [16541]

Внесены изменения в настройки полей для стандартных карточек редактирования.

Теперь в окне настройки карточки редактирования для полей пользователя в качестве имени закладки (столбец Закладка) можно указывать произвольное название. В этом случае на карточке будет создана закладка с этим названием и на нее помещено поле. Также в качестве имени закладки для пользовательского поля можно указать закладку, подключенную в качестве закладки карточки.

 

Сервис

 

[ * ] [16545]

Доработан режим Объединение субконто (вызывается из меню Сервис - Объединение субконто).

При выборе опции Удалять старое значение из списка субконто в том случае, когда субконто используется при формировании оперативных остатков, контроль удаления субконто не позволял удалить строку из справочника.

 

Сейчас режим Объединение субконто работает следующим образом.

Если субконто используется при формировании остатков, то:

 

•- при выборе опции Удалять старое значение из списка субконто, период Весь:

В таблице Остаток создается новая строчка со ссылкой на новое субконто, значение остатка переносится со старого на новое субконто, строчка со ссылкой на старое субконто из таблицы Остаток удаляется. Значение старого субконто из справочника удаляется.

 

•- при выборе опции Удалять старое значение из списка субконто, период НЕ Весь:

Выдается сообщение "Старое значение удалить невозможно (есть оперативные остатки по замененному субконто). Пересчитайте оперативные остатки и удалите значение непосредственно из справочника". Таблица Остаток неизменна. Значение старого субконто из справочника не удаляется.

 

•- опция Удалять старое значение из списка субконто НЕ ВЫБРАНА, период Весь:

В таблице Остаток создается новая строчка со ссылкой на новое субконто, значение остатка переносится со старого на новое субконто, строчка со ссылкой на старое субконто из таблицы Остаток удаляется. Значение старого субконто из справочника  НЕ удаляется.

 

•- опция Удалять старое значение из списка субконто НЕ ВЫБРАНА, период НЕ Весь:

Таблица Остаток неизменна. Значение старого субконто из справочника НЕ удаляется.

 

Изменения внесены в форму [Стандартная. Объединение субконто], которая входит в файл конфигурации BuhtaConfig.mdb текущего релиза.

 

Зарплата

 

[ * ] [16496]

Добавлено два новых алгоритма для начисления заработной платы: За дни(часы)по виду ЗП Статуса и За часы(дни)по виду ЗП Статуса. В карточке редактирования Шифра ЗП на закладке Начисление/Алгоритм при выборе этих алгоритмов открывается дополнительное поле Виды статусов ЗП для указания списка видов статусов. Новые алгоритмы работают так же, как и прежние Начислено за дни(часы), Начислено за часы(дни), но при определении подразделения, оклада, дат приема-увольнения в них берутся только указанные в этом поле виды статусов.

В карточке редактирования для ЗП статуса Внутреннее совмещение добавлено поле Дата конца - в новых алгоритмах эта дата считается датой увольнения.

Исправления, внесенные в пользовательскую форму [Быстрый ввод начислений по Табелю] включены в конфигурацию Стандартная_Зарплата_201.
