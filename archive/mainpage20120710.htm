Отчеты

 

[ * ] [18669]

Внесены изменения в настройки запросов и алгоритмы расчета для следующих форм бухгалтерской отчетности:

- Бухгалтерский баланс,

- Отчет о прибылях и убытках,

- Отчет о движении денежных средств.

 

•Для всех указанных отчетов унифицирован принцип округления - округляются только значения, входящие в результирующую выборку запроса.  Все арифметические операции в формулах производятся с неокругленными значениями.

 

•Изменился алгоритм формирования запросов для отчетов Отчет о прибылях и убытках и Отчет о движении денежных средств. Теперь данные для показателей извлекаются из таблицы [Проводка] с помощью новых SQL-объектов: процедуры [_ОборотСуммаРуб] (для типов показателей Дебет и Кредит) и функции [_СальдоСчетовСуммаРуб] (для свернутого сальдо).

Эти процедура и функция - упрощенный аналог инструмента "SQL-алгоритм". Упрощения:

   - всегда используется поле [Сумма руб.],

   - все валюты,

   - для детализации по субконто используется только тип Прч, который может присутствовать в 1-ом или 2-ом субконто (выбирается автоматически)

 

•Изменилось использование настройки Доп. условие - теперь это дополнительный фильтр к выборке из таблицы [Проводка].

 

[ * ] [18770]

Модифицированы отчеты Дебиторская и кредиторская задолженность и Расшифровка дебиторской и кредиторской задолженности:

•В формы для составления отчетов Дебиторская и кредиторская задолженность и Расшифровка дебиторской и кредиторской задолженности добавлен флаг Только задолженность организаций. При выставленном флаге отчет работает как и ранее, при сброшенном - показываются задолженности по любому субконто.

•В отчете Дебиторская и кредиторская задолженность  из колонок Возникло обязательств на сумму и Погашено обязательств на сумму исключаются задолженности, возникшие и погашенные в отчетном периоде.

•Для варианта расчета отсрочки Срок оплаты указан в признаке договора можно указать не один вид договора, а список (номер признака должен быть одинаковым для всех видов).

[ * ] [18768]

Модифицированы алгоритмы настройки закладок в карточках редактирования:

•Ранее при настройке закладок в общей настройке справочника или настройке вида снятие флажка Использовать приводило к удалению записи из таблицы [Настройка закладок]. Теперь данные о закладке остаются в базе, и закладку можно вернуть. Изменения внесены в объекты эталона: процедуру [Стандартная.Настройка закладок.Сохранить] и функцию [Стандартная.Настройка закладок].

•Для справочников Организации и Подразделения (меню Справочники - Настройка - Организации и Справочники - Настройка - Подразделения) добавлены настройки на закладки:

- Договор.Получатель,

- Договор.Поставщик,

- Документ.Получатель,

- Документ.Поставщик.

На этих закладках выводится список договоров / документов, где текущие организация / подразделение фигурируют в качестве Получателя / Поставщика.

•Для "ручного" добавления новой закладки разработаны формы Стандартная. Настройка закладок и Стандартная. Настройка закладок. Редактирование, которые включены в файл конфигурации BuhtaSupport.mdb.

Особенности настройки:

В списке выбора таблицы, для которой создается закладка, перечислены стандартные справочники, но можно ввести произвольное имя (например, имя пользовательской таблицы).

В список выбора таблицы, данные из которой будут отображаться на закладке, попадают таблицы, имеющие поля типа Ссылка на указанную родительскую таблицу.

Значение в поле Подпись, которое автоматически формируется как <Таблица>.<Поле> можно изменить.

Новые закладки или новое значение подписи появятся только после перезапуска программы (или при пересохранении настройки вида соответствующего справочника).

[ * ] [18738]

На форму выбора режима экспорта в Excel (Экспорт в Excel) добавлен параметр Тип Excel-файла, позволяющий указать формат создаваемого файла  - xlsx (для Excel 2007 и более поздних версий) или xls (для Excel 2003 и более ранних). Это модернизация дает возможность выгрузить данные свыше 65536 строк (для файлов формата xlsx).

Сводные отчеты по-прежнему экспортируются только в формат xls.
