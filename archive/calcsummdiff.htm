Режим ''Расчет суммовых разниц''

 

Добавлен новый режим расчета суммовых разниц по договорам РАСХОДА. Вызывается из главного меню, раздел "Расчеты".

 

Последовательность работы с режимом:

 

1. Поиск договоров для начисления суммовых разниц.На экране необходимо указать:

•ДАТА   - дата, на которую производится расчет суммовых разниц

•САЛЬДО - настроенное сальдо, по которому собирается информация для начисления суммовых разниц. В настройке сальдо можно указать счета, валюту, субконто и другие параметры, ограничивающие список договоров и документов, по которым идет поиск.

•признак ПОКАЗЫВАТЬ ТОЛЬКО НЕЗАКРЫТЫЕ ДОГОВОРА - если признак проставлен, то в список договоров для начисления суммовых разниц НЕ ПОПАДУТ договора, сальдо по которым на указанную ДАТУ нулевые.

•признак СВОРАЧИВАТЬ СУММОВУЮ РАЗНИЦУ - влияет на алгоритм поиска уже начисленной суммовой разницы, если он проставлен, то поиск идет по одному (свернутому) субсчету, который указывается с экрана, если признак не проставлен, то поиск начисленной суммовой разницы идет по двум субсчетам (с положительной и отрицательной суммовыми разницами)

•ПРОЧИЙ СУБСЧЕТ ДЛЯ ПОЛОЖИТЕЛЬНОЙ СУММОВОЙ РАЗНИЦЫ

•ПРОЧИЙ СУБСЧЕТ ДЛЯ ОТРИЦАТЕЛЬНОЙ СУММОВОЙ РАЗНИЦЫ

•ПРОЧИЙ СУБСЧЕТ ДЛЯ СВЕРНУТОЙ СУММОВОЙ РАЗНИЦЫ

 

2. Нажать кнопку "Загрузить", и тогда на экране появится список договоров для начисления суммовых разниц. В список попадают договора, по которым на указанную ДАТУ в указанном САЛЬДО есть обороты по дебету и кредиту.

Для каждого договора выводятся:

•организация-получатель

•номер договора

•дата договора

•суммы из указанного САЛЬДО в рублях:

входящее сальдо

оборот по дебету

оборот по кредиту

исход.сальдо

уже начисленная суммовая разница положительная

уже начисленная суммовая разница отрицательная

В этом списке необходимо отметить (клавишей пробел в квадратике "Оk" или нажатием кнопок "Отметить все"/"Убрать все отметки" под списком договоров) те договора, для которых надо начислить суммовую разницу.

 

3. Укажите параметры для создания документов с суммовыми разницами:

•ВИД ДОКУМЕНТА   - вид создаваемого документа

•НОМЕР ДОКУМЕНТА - номер первого создаваемого документа, начиная с которого будут нумероваться документы, может принимать значение <Авто>, если настроена автонумерация документов этого вида

•ВИД ДОКСПЕЦ - вид докспец

•признак ПРОИЗВОДИТЬ ГЕНЕРАЦИЮ ПРОВОДОК - если признак проставлен, то для каждого созданного документа по спецификациям будет произведена генерация проводок

•кнопка НАСТРОЙКА ГЕНЕРАЦИИ - для просмотра и корректировки настройки генерации проводок в документе.

 

4. Нажать кнопку "Создать".

 

В результате работы режима создаются документы указанного вида. В каждом документе проставлена организация-получатель, совпадающая с организацией, которая проходит одним из субконто в сальдо по договору. Суммовые разницы по договорам объединяются в документы именно по этой организации.Суммовые разницы по договору начисляются отдельными спецификациями на каждый день оплаты по договору (а не на каждую оплату, как в БСофте - это связано с тем, что в Бухте суммовые разницы начисляются по САЛЬДО). При этом производится проверка, не начислена ли уже суммовая разница (по указанным на экране ПРОЧИМ СУБСЧЕТАМ ДЛЯ СУММОВОЙ РАЗНИЦЫ), и начисляется только суммы, превышающие уже начисленную.

 

Режим работает на основе SQL-процедуры dbo.[Создание документов с сумм.разницами].
