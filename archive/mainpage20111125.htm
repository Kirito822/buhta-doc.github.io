Документы

 

[ + ] [18215]

В соответствии с письмом ФНС N ЕД-4-3/15927@ от 28.09.2011 разработан механизм составления корректировочного счета-фактуры (КСФ), в связи с чем произведены следующие изменения:

1.В карточку редактирования документа добавлено новое поле Исходный счет-фактура - ссылка на исходный счет-фактуру.
2.Для ввода и отображения записи таблицы Докспец разработана пользовательская форма [Стандартная. Карточка докспец корр.сч-фактуры], в которой могут редактироваться исходные и новые поля количества, цены и сумм.

3.        Для хранения информации о корректировках в таблице Докспец отведены поля для значений до и после изменений. Поля Количество, Цена, Сумма без налогов, Сумма, Сумма НДС, Сумма всего используются для хранения НОВЫХ значений. Для хранения исходных значений используются поля:

- Количество 2 – Количество исх.

- Ставка таможенной пошлины - Цена исх.

- Нетто - Сумма без налогов исх.

- Сумма возвратной тары  - Сумма НДС исх.

- Сумма амортизации тары  - Сумма всего исх.

4.Добавлена форма [Стандартная. Корректировка счета-фактуры] - вызывается из списка документов по клавишам CTRL+K - предназначена для создания документа вида, специально выделенного под корректировочные счета-фактуры: копирует выбранный документ вместе со спецификациями, в спецификациях нового документа заполняет поля для хранения исходных значений, в новом документе заполняет поле Исходный счет-фактура.
5.Добавлен шаблон отчета [Счет-фактура корректировочный] - вызывается из списка документов (режим Печать по шаблону контекстного меню или по клавише F10).

 

При обновлении версии требуется синхронизации базы данных эталоном текущей версии (в таблицу Документ добавляется новое поле Исходный счет-фактура) и импорт файла конфигурации BuhtaConfig.

 

Примерный план настройки и составления корректировочного счета-фактуры.

 

•Выделить отдельный вид документа для корректировочного счета фактуры. Для этого вида документа создать:

- специальный вид спецификации Корректировка (название примерное), для которого в качестве карточки редактирования указать форму [Стандартная. Карточка докспец корр.сч-фактуры].

-  три вида проводок. Примерные названия: Сумма НДС, Сумма без НДС, Сумма с НДС.

•В списке документов выделить документ с исходным счетом-фактурой подлежащей корректировке. Вызвать форму [Стандартная. Корректировка счета-фактуры] (нажатием CTRL+K). Указать на форме для создаваемого документа значения номера, даты и  вида документа, вида спецификации, а также значение признака Проставить в корректировочном счете-фактуре признак "Без пересчета НДС".

•Открыть созданный документ, перейти на закладку Спецификация, открыть карточку редактирования спецификации, отредактировать значения сумм, сохранить спецификацию, сохранить документ.

•При правильной настройке генерации проводок после сохранения документа должны создаться проводки на разницу между исходными и новыми значениями сумм.

•Составить отчет Корректировочный счет-фактура: в списке документов выделить вновь созданный документ, нажатием клавиши F10 вызвать режим Печать по шаблону и из списка шаблонов выбрать шаблон [Счет-фактура корректировочный].

 

Расчеты

 

[ * ] [18269]

Изменен принцип блокировки таблицы Проводка во время работы режима Расчеты - Закрытие (открытие) периода.

Добавлена новая таблица [Сальдо_НоваяОперативнаяДата] с полями [Юр.лицо], [Дата], в которую в начале работы режима закрытия периода записывается новая оперативная дата. При попытке добавления, изменения и удаления проводки во время работы режима закрытия периода производится сравнение даты проводки с датой указанной в таблице [Сальдо_НоваяОперативнаяДата]), и если дата проводки меньше, выдается сообщение о запрете изменений. После окончания работы режима Закрытие периода запись из таблицы [Сальдо_НоваяОперативнаяДата] удаляется. Напоминаем, в обычной ситуации сравнение дат проводки осуществляется с датой из таблицы СальдоДата.

Модернизация требует синхронизации базы эталоном текущей версии. В эталоне:

- добавлена таблица [Сальдо_НоваяОперативнаяДата] с полями [Юр.лицо], [Дата];

- исправлена процедура [Новая оперативная дата];

- исправлены триггеры [Проводка Trigger Check Saldo_Добавление], [Проводка Trigger Check Saldo_Удаление], [Проводка Trigger Check Saldo_Изменение].
