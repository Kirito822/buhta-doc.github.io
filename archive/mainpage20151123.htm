Справочники

 

[ * ] [21258]

Изменен алгоритм поиска ТМЦ при сканировании по штрих-коду. Теперь запись ищется с помощью функции [скл_Получить_ТМЦ_по_штрих_коду], в которой можно варьировать алгоритм поиска. Сейчас он осуществляется по совпадению значения в поле Номер таблицы Штрих-код. Если запись найдена, то:

- в стандартном списке, построенном по таблице ТМЦ, курсор будет установлен на соответствующем ТМЦ;

- в прайс-листе, в режиме выбора, курсор устанавливается на найденном ТМЦ и одновременно увеличивается количество выписанного товара на то значение, которое прописано в поле Количество таблицы Штрих-код.

В рамках текущей модернизации в эталон БД добавлена функция [скл_Получить_Количество_по_штрих_коду_ТМЦ]

и изменена функция [скл_Получить_ТМЦ_по_штрих_коду].

 

Сервис

 

[ + ] [21188]

Модернизирован режим Сервис - Поиск в объектах:

•Добавлена возможность отметки найденных  объектов  (с помощью клавиши <Пробел> или левой кнопки мышки, а также кнопок <Отметить все>, <Снять все отметки> расположенных в правом верхнем углу окна).

•В окно просмотра найденных объектов добавлен столбец Группа.

•Добавлена возможность экспорта отмеченных объектов (кнопка <Экспорт> в правом верхнем углу окна).

•Добавлена возможность редактирования SQL  объектов.

Напоминаем, что после выхода из режима редактирования в окне Поиск в объектах следует пересоставить список - повторить заново поиск для получения актуальной информации.

[ - ] [21301]

Изменена форма [Стандартная. Составление платежных поручений] (входит в состав Стандартной конфигурации BuhtaConfig). Исправлены следующие ошибки:

•При составлении платежных поручений с типом Подоходный налог и проставленным признаком Создавать докспец с суммами перечислений по каждому сотруднику при пустом значении в поле Вид докспец наравне со спецификациями по сотрудникам создавалась спецификация по общей сумме с пустым значением  в поле Вид.

•Для спецификаций по сотруднику значение поля Входит в сумму определялось не из настроек вида спецификации указанного в настройке платежного поручения Вид докспец Перечисление по сотруднику, а из настроек вида спецификации указанного в поле Вид докспец.
