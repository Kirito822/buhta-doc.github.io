Отчеты

 

[ * ] [19763]

В связи с Приказом  Росстата  от  26.06.2013 N 232 вводится новая форма N 5-З (форма статистического наблюдения), которая начинает действовать с 1-го квартала  2014 года.

Так как в новом варианте отчета изменен состав показателей, то отчеты за периоды начиная с 1 квартала 2014  используют новые запросы [Стандартный. Форма 5-З_2014/1] и [Стандартный. Форма 5-З_2014/2], соответственно за отчетный период и соответствующий период предыдущего года.

 

Настройка показателей для этих запросов может быть произведена следующим образом:

•Если отчет ни разу не составлялся за периоды предшествующие 1-му кварталу 2014 года, то воспользовавшись пользовательской формой Импорт настроек ФО, следует произвести импорт показателей из файла Пример_настройки_ФО.mdb, который входит в состав поставки текущей версии. Нужно на форме Импорт настроек ФО в поле Код формы указать значение 5-З, если требуются показатели для отчета за период ранее 1 квартала 2014, и значение 5-3_2014 - если за 1 квартал 2014 года и позднее. Далее в режиме настройки отчета необходимо откорректировать настройку импортированных показателей и обязательно сохранить запросы при выходе из режима настройки показателей.

•Если в таблице [_Настройка форм отчетности] имеются показатели с кодом формы 5-З и отсутствуют показатели с кодом 5-З_2014, то при вызове режима настройки показателей за период начиная с 1 квартала 2014 года (текущий и период предыдущего года), пользователю будет предложено скопировать настройку запроса за 2014 год из настройки запроса до 2014 года.

- В случае положительного ответа в таблице [_Настройка форм отчетности] будут созданы новые показатели с кодом формы 5-З_2014, совпадающие с уже имеющимися показателями с кодом формы 5-З. Пользователю следует в режиме редактирования показателей откорректировать их в соответствии с новым вариантом отчета.

- В случае отрицательного ответа, следует произвести импорт из файла Пример_настройки_ФО.mdb для кода формы 5-З_2014 и далее в режиме редактирования показателей откорректировать их в соответствии с новым вариантом отчета. Не забывать при выходе из режима редактирования показателей сохранять запросы.

[ * ] [19880]

На форму задания параметров для отчета Кассовая книга (меню Денежные средства - Кассовая книга) добавлен новый параметр Реквизиты юр.лица, организации - ссылка на таблицу Юр.лицо. Параметр используется для заполнения в отчете указанных ниже реквизитов юр.лица и организации, соответствующей этому юр.лицу:

 

- Наименование юр.лица или организации -  в зависимости от значения общей настройки В качестве названия юр.лица использовать,

- Руководитель организации - данные берутся из поля [Руководитель] таблицы Юр.лицо,

- Должность - данные берутся из поля [Должность руководителя] таблицы Юр.лицо,

- Код по ОКПО - данные берутся из поля [ОКПО] таблицы Организация.

 

При начальной установке новой версии параметру Реквизиты юр.лица, организации присваивается значение текущего юр.лица. Выбранное пользователем значение параметра сохраняется в реестре. При пустом значении параметра Реквизиты юр.лица, организации реквизиты заполняются по текущему юр.лицу.

 

Книга покупок и книга продаж

 

[ * ] [19808]

В режиме просмотра Книги покупок и Книги продаж (меню Данные - Книга покупок, Данные - Книга продаж) изменен алгоритм контроля ошибок - добавлена проверка соответствия сумм в колонках:

- Стоимость без НДС 18% и Сумма НДС 18%,

- Стоимость без НДС 10% и Сумма НДС 10%,

- Стоимость без НДС 20% и Сумма НДС 20%.

В колонку Ошибка выводится значение [Стоимость без НДС 18%] *0.18 - [Сумма НДС 18%], если оно не ноль, иначе значение [Стоимость без НДС 10%] *0.10 - [Сумма НДС 10%], если оно не ноль, иначе значение [Стоимость без НДС 20%] *0.20 - [Сумма НДС 20%], если оно не ноль, иначе, как и раньше - значение [Всего покупок с НДС] минус суммы без НДС минус суммы НДС.

 

Модернизация требует синхронизации базы данных эталоном текущей версии для обновления процедур [Книга продаж показать результат] и [Книга покупок показать результат].

 

Безопасность, пароли, доступы

 

[ * ] [19755]

В карточку копирования учетной записи пользователя (режим Настройка - Пользователи / Пароли / Доступы, кнопка Копировать) добавлен параметр Копировать параметры пользователя. Если параметр выставлен, то при копировании пользователя копируются значения следующих полей из карточки учетной записи (см. режим Добавить или Изменить):

- Сотрудник,

- Подразделение,

- Форма,

- Принтер этикетки 104*60,

- Принтер этикетки 50*70,

- Принтер A4,

- Терминалы.

По умолчанию параметр Копировать параметры пользователя выставлен.
