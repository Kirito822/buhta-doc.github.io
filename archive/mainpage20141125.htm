ОС и НМА

 

[ * ] [20602]

Откорректирован алгоритм составления отчета Список ОС с износом  для работы с ОС, которые учитываются на нескольких подразделениях (вызов из меню Отчеты – Учет ОС и НМА - Список ОС с износом).

В случае, если в параметрах отчета выбраны ВСЕ подразделения и материально ответственные лица, и отсутствует любая группировка по ним, ОС в отчет попадает один раз с суммарной балансовой стоимостью и общей суммой износа, в противном случае ОС включается в отчет для каждого подразделения, за которым оно закреплено, с соответствующей балансовой стоимостью и суммой износа, рассчитанной пропорционально этой стоимости.

Обновлена процедура эталона [ОС_Инфо_Выбранные_Валюта_Счет_Подр_МОЛ] - модернизация требует синхронизации БД эталоном текущей версии. Изменены запрос [Стандартный. Список ОС с износом] и скрипт к нему, которые входят в файл конфигурации BuhtaConfig.

 

Справочники

 

[ * ] [20628]

Изменен алгоритм проверки штрих-кодов при вводе: 1) через карточку редактирования справочника ТМЦ, закладка Штрих-код; 2) через форму Ввод штрих-кода, вызывается из списка ТМЦ по горячим клавишам Alt+B или через контекстное меню, пункт Сервис – Ввод штрих-кода:

- Если штрих-код привязывается к ТМЦ, у которого заполнен поставщик, проверка идентичности штрих-кода производится в рамках ТМЦ этого поставщика. Если поставщик не указан, то проверка осуществляется в рамках всего справочника ТМЦ.

- Можно вводить штрих-код длиной 50 символов, ранее не более 13.

 

Отчеты

 

[ * ] [20623], [20625]

Согласно Приказу от 27 августа 2014 г. N 535, начиная с января 2015 года,  вводится новая форма статистической отчетности П-3. Изменены следующие объекты стандартной конфигурации BuhtaForm:

- шаблон [Стандартный. Сведения о фин.сост. орг-ции П-3 2014],

- форма [Стандартный. Сведения о фин.сост. организации П-3]

Внесены также изменения в выгрузку отчета в файлы формата XML – исправлена форма [Стандартная. Архивный отчет в эл.виде (Стат.формы)] из конфигурации BuhtaForm.

 

[ * ] [20630]

Внесены следующие изменения в форму статистической отчетности П-1:

•Откорректирован запрос [Стандартный. Форма П-1/ОКЕИ], отвечающий за Раздел 5 отчета. Запрос дается в качестве примера. Пользователю необходимо самостоятельно настроить значения полей: [s5_0_А], [s5_0_Б], [s5_0_В], [s5_0_Г], [s5_0_Д], [s5_0_1], [s5_0_2], [s5_0_3], [s5_0_4], [s5_0_5] и для параметра запроса Сальдо/Оборот выбрать из предлагаемого списка и установить значение [Стандартный. Форма П-1/ОКЕИ].

•Исправлена форма [Стандартная. Архивный отчет в эл.виде (Стат.формы)], отвечающая за выгрузку отчета в файлы формата XML.

Измененные объекты входят в стандартную конфигурацию BuhtaForm.

АРМ

 

[ * ] [20654]

При вызове списка из АРМ можно просмотреть данные по виду, не указанному в  настройке АРМ, заведя в поле Вид номер вида вручную.

Подобное несоответствие устранено для следующих форм просмотра:

- Договоры,

- Документы,

- Организации,

- ТМЦ,

- Сотрудники,

- Подразделения,

- ОС,

- Трудовые договоры,

- Приказы

При указании вида отсутствующего в списке видов, доступных для просмотра, выдается сообщение: "Указанный вид ... не существует или не доступен для просмотра".
