Справочники

 

[ + ] [14638]

Модернизирован режим настройки иерархии справочников (меню Справочники - Настройка - Иерархия в справочниках) - добавлена настройка иерархии по виду или по полю справочника.

 

Для настройки иерархии по полю у соответствующего справочника должен быть установлен флаг в столбце Иерархия и снят флаг в столбце По виду.

Значение в поле По полю по умолчанию Номер. При такой настройке справочник отображается в том же виде, что и в предыдущих релизах - список занимает полностью все рабочее окно.

При указанной выше настройке пользователь в праве выбрать иерархию по любому символьному полю справочника. Для этого следует выбрать и установить нужное значение в поле По полю. Если это значение отличается от значения Номер, то в этом случае справочник отображается в окне состоящем из двух частей. В левой части представлено дерево иерархии, в правой - элементы той ветви, на которой установлен курсор.

Основной функционал этого режима представлен в контекстном меню списков.

Помимо этого в правой части окна существует возможность фильтрации по виду - поле Вид.

Для левой части окна может быть задействована кнопка на Главной инструментальной панели Фильтр по иерархии вкл/выкл. В зависимости от состояния этой кнопки в левой части окна отображаются только собственно элементы выделенной ветви дерева иерархии или также все подчиненные элементы этой ветви.

Также в левой части окна методом перетаскивания (drag-and-drop) можно переместить элемент из одного узла дерева иерархии в другой.

Для настройки иерархии по виду у соответствующего справочника должен быть установлены флаги в столбце Иерархия и в столбце По виду, какое значение при этом в поле По полю - безразлично. При такой настройке справочник также отображается в окне состоящем из двух частей. В левой части - справочник видов, в правой - элементы того вида, на котором установлен курсор в левой части.

Отчеты

 

[ + ] [14703], [14706], [14924]

В целях унификации подхода к составлению отчетности, основанной на выборках из сальдо и проводок, разработана новая настройка для форм отчетности. Принцип составления отчетности следующий

- один раз выполняется формирование Сальдо/Оборота по необходимому перечню счетов для всего отчета

- по настройке, для каждой ячейки отчета выбираются показатели - например, суммы сальдо или оборотов по части счетов(субсчетов).

Для сохранения настроек для подобных отчетов в базу добавлена таблица [_Настройка форм отчетности] и разработаны функции, позволяющие при составлении отчетов выполнять выборку настроек - значений из указанной таблицы.

 

В рамках этого алгоритма  разработаны следующие новые отчеты статистического наблюдения:

- СВЕДЕНИЯ О ПРОИЗВОДСТВЕ И ОТГРУЗКЕ ТОВАРОВ И УСЛУГ, форма № П-1

- СВЕДЕНИЯ ОБ ОБЪЕМЕ ПЛАТНЫХ УСЛУГ НАСЕЛЕНИЮ ПО ВИДАМ, приложение № 3 к форме № П-1

и переделаны отчеты:

- СВЕДЕНИЯ О СОСТОЯНИИ РАСЧЕТОВ ЗА ОТГРУЖЕННУЮ ПРОДУКЦИЮ, ВЫПОЛНЕННЫЕ РАБОТЫ (УСЛУГИ), форма № 1-РП

- СВЕДЕНИЯ О ФИНАНСОВОМ СОСТОЯНИИ ОРГАНИЗАЦИИ, форма № П-3.

Все эти отчеты запускаются из пользовательских форм.

 

Также добавлены новые варианты (формы) составления отчетов для уже имеющихся отчетов из меню Отчеты - Формы бухгалтерской отчетности:

- БУХГАЛТЕРСКИЙ БАЛАНС, код формы Ф1

- ОТЧЕТ О ПРИБЫЛЯХ И УБЫТКАХ, код формы Ф2

- ОТЧЕТ О ДВИЖЕНИИ ДЕНЕЖНЫХ СРЕДСТВ, код формы Ф4

Для того чтобы отличить их от прежних, к названиям новых форм и связанных с ними объектов  добавлен постфикс 91 (номер релиза, начиная с которого можно работать с новыми формами указанных отчетов).

 

Примеры настроек отчетов содержатся в следующих файлах формата mdb в каталоге Дополнения:

- для форм статистического наблюдения - в файле Пример_настройки_ФО.mdb,

- для форм бухгалтерской отчетности - в файле Пример_настроек_Ф1_Ф2_Ф4.mdb.

 

Для импорта/экспорта файла в таблицу/ из таблицы [_Настройка форм отчетности] разработаны новые пользовательские формы Импорт настроек ФО и Экспорт настроек ФО.

 

Помимо этого для отчетов типа 2 (MS Excel) разработана система сохранения показателей отчетов прошлых периодов  в таблицу [_Показатели из архивных отчетов]. В структуру таких, а именно, всех вышеперечисленных отчетов добавляется лист с названием Показатели, содержащий список сохраняемых показателей, которые можно также использовать при настройке  указанных отчетов.

 

Все добавленные и измененные объекты входят в конфигурацию BuhtaForm.mdb

Внесены следующие изменения в процедуру удержания алиментов:

1.За месяц, в котором ребенку исполняется 18 лет, алименты удерживаются частично, по следующей формуле:

< Алименты за полный месяц> х <Число дней в месяце до дня совершеннолетия ребенка>/<Число дней в месяце>.

Например, дата рождения ребенка, на которого уплачиваются алименты, - 19 июля. Полная сумма алиментов с дохода за июль составляет 4000 руб. Тогда с дохода за июль с работника нужно удержать алименты в размере 2452 руб. (4000 руб. x 19 дн. / 31 д.).
2.При расчете максимальной суммы алиментов и по исполнительному листу  из суммы начислений вычитается не только сумма подоходного налога, но и сумма всех шифров, которые указаны в Привязке шифров шифра Алименты с проставленным признаком Исключить период.

 

[ * ] [15196]

В процедуру [Заполнение Ставка по дате] добавлен новый тип ставки налога Мат.выгода от ставки рефинан. и новые ставки с этим типом: с 1 января 2000 = 0.75 (три четвертых), с 1 января 2009 = 0.6667 (две третьих).

Теперь при расчете мат.выгоды Ставка рефинансирования умножается не на 0.75, как раньше, а на значение, которое вычисляется по таблице Ставка на дату по типу Мат.выгода от ставки рефинан. на конец расчетного месяца з/п. Если это значение =0 или не найдено, то на дату после 1 января 2009 присваивается значение = 0.6667 (2/3), а раньше = 0.75 (3/4).

•Добавлена возможность удвоения минимума по отказу от вычета:

- В карточку редактирования записи о составе  семьи (вызывается на закладке Дети кадровой карточки Сотрудника) добавлен признак По отказу второго родителя от получения вычета.

- В карточку налоговых данных сотрудника (вызывается на закладке Налоги личной карточки Сотрудника) в список возможных значений для поля Удвоение минимумов добавлено значение Да (отказ).

 

•Начиная с 2009 года:

вместо вычета 101 используется вычет 108,

вместо вычета 106 используется вычет 109,

вместо вычета 102 (удвоение) используется вычет 110 или вычет 111 (если удвоение по отказу),

вместо вычета 107 (инвалидность и удвоение) используется вычет 112 или 113 (если инвалидность и удвоение по отказу).
