Прайс-лист

 

[ * ] [17673]

В карточку настройки Прайс-листа (меню Данные - Настройка - Прайс-лист) на закладку Дополнительно добавлен новый признак Не восстанавливать цены по умолчанию для выбранных позиций  (поле Не менять цены для выбранных ТМЦ в таблице Прайс). Если признак выставлен, то при наличии выбранных товаров обновление Прайса по кнопке <Выборка> не меняет значения цен и сумм у выбранных товаров.

 

[ + ] [17719], [17758]

В режим настройки Прайс-листа (меню Данные - Настройка - Прайс-лист):

• Добавлен новый тип быстрого поиска Список по фрагментам. При этом варианте поиске вводимые пробелы рассматриваются как условие И (and). Например, при задании значения "AA BB" будут найдены значения "AA BB", "BB AA","AABB", "AACCBB", "BBAA", "BBCCAA".

•Для вариантов поиска Список... добавлен признак Первичный поиск - внутри поля группировки. При установке этого признака выполняется  поиск по заданным условиям внутри поля группировки. Если не найдено ни одной позиции, то выполняется поиск по всему списку. В форму с результатом поиска добавлена  кнопка Продолжить по всему списку. Кнопка доступна, если поиск по виду был успешным. По ее нажатию выполняется  поиск по всему списку.

 

[ - ] [17729]

В режиме Данные - Прайс-лист исправлена ошибка обновления данных при выборе товара при одновременной работе нескольких пользователей.

 

Отчеты

 

[ + ] [17660]

Добавлена возможность пользовательской сортировки в списке шаблонов для печати. Для этого используется SQL-функция, в которую в качестве параметров передаются Имя пользователя, Таблица и Вид. Имя функции указывается в общей настройке приложения (меню Настройка - Общая настройка), на закладке Внешний вид, в параметре SQL - Функция сортировки шаблонов печати. В список возможных значений для этого параметра отбираются функции, название которых имеет окончание ". Список шаблонов для печати" - перед точкой может быть любой префикс.

В качестве примера в стандартную конфигурацию BuhtaConfig включена процедура [Стандартная. Список шаблонов для печати], которая сортирует список шаблонов в том же порядке, что и  в прежних релизах.

Если значение для параметра SQL - Функция сортировки шаблонов печати не назначено, список шаблонов для печати формируется по тому же алгоритму, что и в прежних релизах.

 

[ +] [17661]

Внесены дополнительные возможности для настройки Сводного отчета (меню Отчеты - Сводный отчет).

В настройку областей строк и колонок отчета в список полей добавлена колонка Сортировка, позволяющая выбрать один из алгоритмов сортировки:

- по возрастанию: данные типа Дата упорядочиваются по возрастанию значения даты,  строковые данные - по возрастанию значения символьных строк;

- по убыванию: данные типа Дата упорядочиваются по убыванию значения даты, строковые данные - по убыванию значения символьных строк;

- по возрастанию (дата): данные типа Дата упорядочиваются по возрастанию значения даты, строковые данные - по возрастанию значения даты, содержащейся в строке после символов "от";

- по убыванию (дата): данные типа Дата упорядочиваются по убыванию значения даты, строковые данные - по убыванию значения даты, содержащейся в строке после символов "от";

- по возрастанию (номер): данные упорядочиваются по возрастанию числового значения строки, содержащейся в квадратных скобках;

- по убыванию (номер): данные упорядочиваются по убыванию числового значения строки, содержащейся в квадратных скобках.

 

[ + ] [17692]

Внесены следующие изменения в режимы формирования отчетов Дебиторская и кредиторская задолженность (меню Отчеты - Расчеты с дебиторами и кредиторами - Дебиторская и кредиторская задолженность) и Расшифровка дебиторской и кредиторской задолженности (меню Отчеты - Расчеты с дебиторами и кредиторами - Расшифровка дебиторской и кредиторской задолженности).

 

Для каждого из этих отчетов на форму задания параметров:

•добавлены  варианты определения отсрочки оплаты для дебиторов, т.к. отсрочка может быть разной не только для разных организаций, но и для разных договоров. Отсрочка определяется в следующей последовательности:

   - дата оплаты может быть указана в признаке договора;

   - если не указана, количество дней отсрочки указывается в специальном поле договора;

   - если не указано, количество дней отсрочки берется из карточки организации (поле Отсрочка);

   - если поле не заполнено, берется количество дней отсрочки, указанное в поле формы.

•добавлен параметр Просматривать историю возникновения задолженности за <...> месяцев (значение по умолчанию 12 месяцев). В прежних версиях этот параметр был равен 24 и жестко задавался при формировании отчета, без возможности изменения.

 

Для отчета Расшифровка дебиторской и кредиторской задолженности:

•  На форму добавлен признак С детализацией по договорам, позволяющий составить отчет с указанием суммы задолженности по каждому договору.

• Если отчет составляется в тысячах денежных единиц, задолженности дебиторов и кредиторов, в сумме не превышающие 1000 ед., складываются и заносятся в строку Прочие задолженности.

 

[ + ] [17759]

В соответствии с Приказом Минфина России от 02.07.2010 N 66н утверждены новые формы Бухгалтерского баланса и Отчета о прибылях и убытках. В связи с этим разработана новая пользовательская форма [Стандартная. Бухгалтерский баланс 2011], которая позволяет составлять оба отчета Бухгалтерский баланс и Отчет о прибылях и убытках в виде одного Excel-файла.

Все объекты, необходимые для построения этих отчетов, включены в файл Стандартной конфигурации BuhtaForm.mdb.Сюда же входят обновленные  формы [Стандартная. Архивный отчет в электронном виде] и [Стандартная. Архивный отчет в электронном виде_91], отвечающие за выгрузку в XML файл отчета, хранящегося в архиве под наименованием Стандартный. Бухгалтерский баланс 2011.

В поставку текущей версии также входит обновленный файл Пример_настройки_ФО.mdb, в который добавлены строки с примерами  настроек для новых форм Бухгалтерского баланса (код формы Ф1_2011) и Отчета о прибылях и убытках (код формы Ф2_2011).

 

Пользователю через конфигуратор следует произвести импорт файла BuhtaForm.mdb, перезагрузить приложение. Затем, воспользовавшись пользовательской формой Импорт настроек ФО, закачать данные для форм Ф1_2011 и Ф2_2011, запустить форму [Стандартная. Бухгалтерский баланс 2011] и приступить к настройке отчетов.

 

Существенные отличия в настройке отчетов по сравнению с предыдущей версией заключаются в следующем:

- добавлены настройки запросов за предыдущий период,

- добавлена возможность сохранения рассчитанных показателей в таблицу [_Показатели из архивных отчетов] и их дальнейшего использования для настройки запросов за предыдущие периоды.

 

При нажатии на кнопку расположенную справа от параметра Составлять Бухгалтерский баланс вызывается настройка запросов для Бухгалтерского баланса, справа от параметра Составлять отчет о прибылях и убытках  - настройка запросов для Отчета о прибылях и убытках.

 

Для составления Бухгалтерского баланса необходимо настроить два запроса:

- за отчетный период,

- за предыдущие два года.

 

Для составления Отчета о прибылях и убытках необходимо также настроить два запроса:

- за отчетный период,

- за соответствующий период предыдущего года.

 

Напоминаем, что после изменения настроек показателей или их импорта с помощью формы Импорт настроек ФО, следует при выходе из окна со списком настроек нажать на кнопку <Сохранить>. В противном случае список настроек  не будет соответствовать запросу, по которому составляется отчет.

 

Рекомендации по настройке запросов.

Запросы за предыдущие годы можно рассчитывать по сальдо оборотам, выбрав в карточке настройки для параметра Тип сальдо  значения Дебетовое Сальдо, Кредитовое сальдо, Свернутое сальдо, для параметра Тип оборотов - значения Дебет, Кредит, а можно брать из таблицы [_Показатели из архивных отчетов], в этом случае для параметра Тип сальдо/оборотов указывается значение Показатели из отчетов.

При этом для настройки Бухгалтерского баланса из таблицы [_Показатели из архивных отчетов] по каждой строке выбираются значения [Ф1_2011 рзд.1 стр.<номер строки> грф.1] для формы Ф1_2011, а для Отчета о прибылях и убытках по каждой строке выбираются значения [Ф1_2011 рзд.2 стр.<номер строки> грф.1] для формы Ф2_2011.

Воспользоваться таблицей [_Показатели из архивных отчетов] можно в том случае, если она содержит нужные показатели за предыдущие годы. Для этого достаточно составить Бухгалтерский баланс за 2009  и 2010 год, а Отчет о прибылях и убытках за 1 квартал 2010 года, 1 полугодие 2010 года, 9 мес 2010 года и 2010 год, и при сохранении этих отчетов, на вопрос о сохранении данных в таблицу показателей ответить Да.

 

Формы

 

[ - ] [17666]

При вызове Hot-Кey форм из справочников (контекстное меню, подменю Пользовательские режимы или Сервис - Обработчик, или "горячие клавиши" Alt+F12), в том случае если справочник был отфильтрован по виду, при формировании списка доступных Hot-Кey форм не учитывалась настройка Hot-Кey формы на вид таблицы - выводились все Hot-Кey формы, относящиеся к соответствующей таблице безотносительно к ее виду. Недочет устранен. Теперь в список включаются только те Hot-Кey формы, в настройке которых или явно указан  вид таблицы, для которого этот список вызывается, или значение Все, или пусто.

Синхронизация

 

[ + ] [17322]

В режиме синхронизации баз данных, в форму задания параметров на Шаге 3 добавлен флаг Не выводить сообщения об ошибках и диалоги.

Указанный флаг при выставлении позволяет произвести синхронизацию (после того как будет нажата кнопка <Запустить>) без вмешательства пользователя.

При этой настройке:

- сообщения об ошибках на экран выводиться не будут, прочитать их можно будет только в соответствующем log - файле результатов синхронизации;

- ситуации, разрешаемые с помощью диалоговых сообщений, будут решаться так, как если бы пользователь ответил Нет; описание принятия подобных решений и их последствий также будет зафиксировано в log-файле.
