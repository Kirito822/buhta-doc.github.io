Расчеты

 

[ - ] [16752]

Внесены изменения в режим Расчеты - Закрытие счета. При закрытии счета пропорционально видам спецификации документа (на форме задания параметров выставлен флаг Закрыть пропорционально) в поле Текст документа записывался соответствующий комментарий. При большом списке спецификаций этот комментарий был достаточно велик, что приводило к ошибке создания документа. Недочет устранен - теперь в поле Текст документа записывается только 330 первых символов создаваемого комментария.

 

Кадры

 

[ + ] [16571]

Добавлена возможность установки системы блокировки на режим работы с Табелем учета рабочего времени (меню Кадры). Блокировка настраивается в режиме Настройка - Блокировки - Табель и производится в рамках текущего юр.лица по полю Период. Если значение, указанное в этом поле, включает хотя бы одну дату какого-либо месяца, то для этого месяца в режиме Кадры - Табель учета рабочего времени заблокирована возможность сохранения сформированного табеля.

 

Дополнительно в раздел доступов Сервис - Безопасность пароли доступы - Блокировки добавлен доступ Групповая блок. табеля (из меню Настройка). При его отсутствии  режим Настройка - Блокировки - Табель недоступен.

 

Отчеты

 

[ * ] [16568]

В окно Общей настройки системы (режим Настройка - Общая настройка) на закладку Внешний вид добавлен флаг Скрывать в сводных отчетах поле "Период", если таблица, по которой строится отчет, не "Сальдо/Оборот".

 

Действие флага применяется в случае если:

- Отчет строится не по таблице Сальдо/Оборот.

- В отчете отсутствуют дескрипторы ^DialogBegDate^ или ^DialogEndDate^.

 

Ранее при указанных условиях поле Период присутствовало в окне формирования Сводного отчета (режим Отчеты - Сводный отчет), но его значение не оказывало никакого влияния на информацию, выводимую в отчете. Новый флаг позволяет скрыть в окне формирования Сводного отчета ненужные параметры.  

 

[ + ] [16733]

Модифицированы режимы формирования отчетов:

- Акт инвентаризации расчетов ИНВ-17 (вызывается из меню Отчеты - Расчеты с дебиторами и кредиторами - Акт инвентаризации ИНВ-17 или из пользовательской формы [Стандартная. Акт инвентаризации ИНВ-17]),

- Справка к акту ИНВ-17 (вызывается из пользовательской формы [Стандартная. Справка к акту ИНВ-17]).

На форме [Стандартная. Акт инвентаризации ИНВ-17] добавлен флаг Использовать фильтр по организациям. Если он выставлен, то появляется возможность указания фильтра по списку организаций. Если указан список организаций, то в отчет будут отбираться данные только по тем сальдо/оборотам, у которых в субконто 1 с типом Орг указано одно из выбранных значений в списке организаций.

На форме [Стандартная. Справка к акту ИНВ-17] добавлен флаг Использовать фильтр по организациям из акта ИНВ-17. Если он выставлен, на информацию, которые попадает в Справку к акту ИНВ-17, накладывается тот же фильтр, что и в предыдущем сеансе формирования отчета Акт инвентаризации расчетов ИНВ-17.

Безопасность, пароли, доступы

 

[ + ] [16767]

В связи с заменой Единого Социального Налога (ЕСН) на Страховые Взносы (С/В) в права доступа (меню Настройка - Пользователи / Пароли / Доступы) в раздел Зарплата - Зарплата / Режимы добавлено новое право Расчет СВ.

Это право является аналогом права Расчет ЕСН, которое отвечает за расчет Единого Социального Налога, поскольку начиная с января 2010 года алгоритм Расчета ЕСН заменен алгоритмом Расчета СВ.

Администратору системы после обновления версии и синхронизации базы данных новым эталоном следует дать доступ к  праву Расчет СВ всем пользователям, которым оно необходимо.

 

Зарплата См. также раздел Конфигурация ''Стандартная_Зарплата_...''

 

[ - ] [16673]

Исправлены алгоритмы начисления заработной платы:

- Начислено за дни по окладу

- Начислено за часы (дни)

- Начислено за дни (часы)

 

Ранее при расчете заработной платы датой начала расчета по шифру в указанных алгоритмах считалось первое число расчетного месяца (или Дата приема, если она позже первого числа расчетного месяца), и если месяц отработан не полностью, то оклад-тариф-подразделение-вид дохода брались на первое число месяца. Теперь, если в карточке настройки шифра ЗП на закладке Начисление/Карточка проставлен признак Даты начала и окончания, то при расчете заработной платы по ЗП Инфо по этому шифру учитываются даты начала и окончания, указанные в ЗП Инфо, а  именно: если Дата начала в ЗП Инфо больше Даты начала расчетного месяца (или даты приема на работу в этом месяце), то Дата начала расчета шифра приравнивается к Дате начала в ЗП Инфо, и оклад-тариф-подразделение и т.д. берутся на эту дату, то же самое с Датой конца из ЗП Инфо - если она меньше даты конца расчетного месяца (или даты увольнения), то дата окончания расчета шифра приравнивается к Дате конца из ЗП Инфо.

 

[ * ] [16779]

В настройку карточки редактирования Шифра ЗП типа Удержание добавлена закладка Налоги, что в частности позволяет заполнить поле Вид вычета для шифров ЗП по договорам негосударственного пенсионного обеспечения.

Для получения налогового вычета [319] при расчете подоходного налога по таким шифрам следует произвести дополнительные настройки:

1) в случае отсутствия в клиентской базе в таблице Ставки на дату (меню Зарплата - Настройка - Ставки на дату) записи с типом Налоговый вычет и кодом 319 добавить ее со значениями Дата=01.01.2010 и Сумма=120000.

2) во всех ЗП Инфо по шифрам, в карточке которых проставлен вид вычета 319, указать значение поля Ставка подоходного=13.
