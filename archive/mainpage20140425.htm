Отчеты

 

[ * ] [20146]

Исправлена форма [Стандартный. Сведения о фин.сост. организации П-3] (конфигурация BuhtaForm) для построения отчета СВЕДЕНИЯ О ФИНАНСОВОМ СОСТОЯНИИ ОРГАНИЗАЦИИ (Форма № П-3).

Теперь, если в настройке запроса за предыдущий период указан параметр 1_02_1 (Прибыль (убыток) до налогообложения за соответствующий период с начала предыдущего  года), то в отчет в строку 02 раздела 01 заносится значение этого параметра. Если параметр не указан, то как и прежде в отчет попадает значение параметра 1_02_1 из настройки запроса за отчетный период.

Если значение параметра 1_02_1 должно соответствовать значению параметра 1_01_1 (Прибыль (убыток) до налогообложения за период с начала отчетного года) за предыдущий период и при этом выбираться из сохраненных показателей (таблица _Показатели из архивных отчетов), то в настройке запроса за предыдущий период для параметра 1_02_1  следует указать тип настройки Показатели из отчетов со ссылкой на показатель [П3 рзд.1 стр.01 грф.1].

Настройка

 

[ + ] [20168]

В файл настроек BuhtaClient.ini добавлен новый параметр DATABASES - список отображаемых баз данных в загрузочной форме модуля  БУХта.

Для мобильной (секция [MOBILEPARAM]) и полной версии (секция [NOMOBILEPARAM]) указывается свой персональный параметр DATABASES.

Значение параметра DATABASES - перечень имен баз данных указанных через запятую. Если параметр DATABASES пустой или отсутствует, то при вызове загрузочной формы модуля БУХта в ниспадающем списке у поля База отображаются все доступные базы.

 

Зарплата См. также Конфигурация ''Стандартная_Зарплата_...''

 

[ * ] [20240]

В связи с введением в платежное поручение нового реквизита - уникального идентификатора начислений (УИН) внесены следующие изменения:

- В структуру БД - в таблицы Документ и ЗП Перечисление добавлено новое поле ПП.Код УИН. Модификация требует синхронизации БД эталоном текущей версии.

- В карточки редактирования Документа типа Стандартная и Платежное поручение (-) и в карточку редактирования ЗП Перечисления  - добавлено новое поле Код УИН.

 

[ * ] [20249]

В соответствии с Федеральным законом РФ от 2 апреля 2014 г. N 55-ФЗ изменен коэффициент при расчете среднего дневного заработка для оплаты отпусков и выплаты компенсации за неиспользованные отпуска. Начиная с даты начала отпуска равной 02.04.2014 вместо коэффициента 29.4 при расчете используется коэффициент 29.3.

В рамках этой модернизации изменена процедура эталона [З/П Доходы и дни к расчету среднего] и форма [ЗП расчет среднего заработка], входящая в состав конфигурации Стандартная_Зарплата.

 

Информация для разработчиков

 

[ + ] [20149]

В дизайнер форм пользователя добавлен доступ к свойству Scaled. Теперь пользовательские формы создаются "мастером" создания форм по умолчанию со свойством Scaled = false.

Поскольку ранее пользовательские формы создавались по умолчанию со свойством Scaled=true, то после изменения масштаба шрифта, при запуске дизайнера все элементы у таких форм будут отмасштабированы. Их нужно заново выровнять вручную, а затем установить свойство Scaled=false.
