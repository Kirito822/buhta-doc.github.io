Отладчик скриптов

 

Отладчик скриптов БУХта Script Debugger обеспечивает отладку разрабатываемых пользовательских форм и скриптов, значительно упрощает работу программиста.

Запускается как отдельное приложение.

Для отладки работы процедур и функций разработчику предоставляется пошаговый отладчик со стандартным для подобных приложений функционалом: контрольные точки останова, трассировка, просмотр значений переменных и т.п.

 

Для запуска отладчика необходимо чтобы файл dsd.exe находился в той же папке, что и клиентское приложение. Тогда из приложения БУХта его можно запустить с помощью скриптов:

 

1) Автоматически, при старте приложения. Надо добавить скрипт для класса ApplicationEvents – событие Application.OnRun

ShellExecute 0, "open", ExtractFilePath(bm.ExeName) + "dsd.exe", Nil, Nil, SW_SHOW

Этот же код можно использовать и для АРМ'а в VBScript'е.

 

2) Из Pascal-скриптов, с помощью вызова

ShellExecute(0, 'open', ExtractFilePath(bm.ExeName)+'dsd.exe' , Nil, Nil, SW_SHOW);

 

В появившемся окне в меню View выбрать пункт Running Scripts

 

 

В окне Select script to debug будут показаны запущенные на компьютере скрипты.

 

 

Если щелкнуть по одному из них мышью, то откроется окно отладки соответствующего скрипта. В этом окне действуют основные команды отладчика Delphi.

 

 

Можно добавлять/удалять точки останова, производить трассировку кода, добавлять выражения для отслеживания/вычисления их значений. Пока приложение остановлено в режиме отладки, для переменных в контекстном меню отображаются их значения.

 

 

Горячие клавиши:

 

F9...........Выполнить

F5...........Установить или удалить точку останова

F8...........Трассировка без захода в метод или функцию

F7...........Трассировка с заходом в метод или функцию

F4...........Выполнить код до курсора и остановиться

Ctrl+F2...Остановить отладку
