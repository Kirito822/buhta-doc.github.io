Договоры и документы

 

[ * ] [20427]

•Добавлен контроль на заполнение настроек в режиме работы с ТСД. Проверяется заполнение полей Поле докспец/догспец для записи ТМЦ и Поле докспец/догспец для записи количества, а также наличие видов спецификаций 1 и 999. Проверка осуществляется при сохранении вида документа/договора, а также при запуске режима работы с ТСД.

•В настройку видов  документов / договоров добавлено поле Название режима для ТСД. Модернизация требует обновления структуры  базы данных эталоном текущей версии.

 

[ * ] [20446]

В текст сообщения о блокировке для документов/договоров добавлена информация ([ключ документа] + Вид.Номер + Дата) о первом документе/договоре входящем в условие блокировки.

После обновления релиза указанное изменение вступит в силу только после пересоздания триггеров блокировок. Вызов функции пересоздания осуществляется нажатием кнопки <Пересоздать триггеры> на формах со списками блокировок документов и  договоров (меню Настройка - Блокировки - Документы/Договоры).

 

[ * ] [20469]

Изменено сообщение на подтверждение удаления записи из таблиц Догспец / Докспец - в текст сообщения добавлена ссылка на поле Догспец.[Субконто 1].[Номер и название] / Докспец.[Субконто 1].[Номер и название].

 

Для разработчиков:

Для компонента TbuhtaGrid доступно событие OnGetNumNameRecord, с помощью которого можно переопределить сообщение выдаваемой системой.

 

Справочники

 

[ + ] [20373]

На закладку Дополнительные комплекты стандартной карточки редактирования ТМЦ и на закладку Комплектующие карточки редактирования  дополнительного комплекта добавлено поле Период, которое обеспечивает просмотр действующих дополнительных комплектов и соответствующих комплектующих.. При изменении периода в карточке ТМЦ устанавливаются значения двух параметров : Доп. комплект дата начала и Доп. комплект дата конца, которые передаются в скрипт запроса [ТМЦ Комплект Вид] (входит в состав конфигурации BuhtaConfig).

 

Отчеты

 

[ * ] [20439]

Внесены доп.изменения (см. также релиз 155) в алгоритм составления Налоговой декларации по налогу на прибыль организаций (меню Отчеты – Формы налоговой отчетность – Налоговая декларация на прибыль) - на форму предварительного задания параметров новый признак Заполнять суммы ежемесячных авансовых платежей (Раздел 1.2 и Лист 02) (обнуляется, если выбран период ГОД).

 

Если он проставлен, то:

•в Разделе 1.2:

строка 120 = строке 130 = строке 140 = 1/3 строки 300 Листа 02;

строка 220 = строке 230 = строке 240 = 1/3 строки 310 Листа 02.

• на Листе 02 заполняются строки 290-300-310 (из строк 180-190-200 текущего отчета). Если при этом проставлен признак Данные об авансовых платежах - из архивного отчета за пред.период и указан отчет, то для отчетов за 1 полугодие и 9 месяцев строка 290 = строка 180 тек.отчета минус строка 180 пред.отчета (аналогично 300 = 190тек-190пред, 310=200тек-200пред), если значения строк 290-300-310 отрицательные, они обнуляются. Строки 320-330-340 заполняются из строк 290-300-310 только для отчета за 9 месяцев (код периода 33 на титульном листе).

 

Следующие объекты, измененные в рамках текущей модернизации, входят в конфигурацию BuhtaForm:

1. запрос [Стандартный. Налоговая декларация по НалПриб Архив] со скриптом

2. запрос [Стандартный. Декларация по нал. на прибыль 2007] со скриптом

3. форма [Стандартный. Налоговая декл. по налогу на прибыль]

4. шаблон [Стандартный. Нал. декл.по налогу на прибыль 2013]
