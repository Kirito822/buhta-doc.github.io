Договоры и документы

 

[ * ] [19354]

В формах копирования договоров и доверенностей исправлен алгоритм реагирования на нажатие клавиши <Enter>. Ранее при нажатии <Enter> форма копирования закрывалась, и выполнялось копирование. Сейчас нажатие <Enter> приводит к переводу фокуса ввода на следующий элемент.  Это выполняется только в одном случае, если активный элемент - поля для ввода значения, в противном случае ничего не происходит.

 

[ * ] [19381]

Изменены процедуры эталона [dbo].[Заполнение суммы документа], [dbo].[Заполнение суммы договора] - добавлено заполнение полей Кто изменил, Когда изменил.

 

Прайс-лист

 

[ - ] [19319]

Исправлены ошибки обновления экрана Прайс-листа в режиме выбора товара (меню Данные - Прайс-лист).

 

[ - ] [19371]

Начиная с SQL Server 2005 в настройках Прайса для функции раскрытия и функции данных (требуются при использовании 2-х субконто) невозможно было определить число параметров. Ошибка исправлена. Требуется синхронизация эталоном текущей версии.

 

Курсы валют

 

[ * ] [19285]

Режим импорта курсов валют с сайта компании РосБизнесКонсалтинг (вызывается из меню Данные - Курсы валют, кнопка Загрузка курсов) дополнен импортом количества валюты. Исправленная форма [Импорт курсов валют (РБК)] входит в состав файла конфигурации BuhtaConfig.

 

Справочники

 

[ * ] [19240]

Длина поля Страна из справочника ГТД увеличена до 45 символов. В карточку редактирования справочника ГТД (меню Справочники - Прочие справочники - ГТД) добавлена возможность выбора  названия страны из справочника ГНИ код страна.

 

Отчеты

 

[ * ] [19180]

В связи с введением с 1 квартала 2013 года нового варианта отчета РСВ-1 (составляется по шаблону [З/П Расчет по взносам на ОПС и ОМС РСВ-1 ПФР 2013], см. раздел Конфигурация ''Стандартная_Зарплата_...'' ) изменены формы [Стандартная. Архивный отчет в электронном виде] и [Стандартная. Архивный отчет в эл.виде (Дополнения)] из конфигурации BuhtaForm.mdb, которые отвечают за экспорт отчетов в файлы XML -формата.

 

[ - ] [19379]

Исправлена форма [Настройка отчета о прибылях и убытках] (входит в файл конфигурации BuhtaForm.mdb) - в 138 релизе была допущена опечатка - в фразе "IF OBJECT_ID(''tempdb..#NewLine'') IS NOT NULL 0". Ошибка проявлялась  при изменении параметров настройки в Отчете о прибылях и убытках.

 

Зарплата

 

[ + ] [19313]

С 1 января 2013 года введены дополнительные тарифы страховых взносов для отдельных категорий страхователей (статья 33.2, введена Федеральным законом от 03.12.2012 N 243-ФЗ). В связи с этим  в систему расчета страховых взносов комплекса БУХта внесены следующие изменения:

•В форму настройки зарплаты (меню Зарплата - Настройка - Основная настройка) на закладку Страх.взносы добавлены новые параметры: Виды докспец С/В: ОПС доп.тариф и Перечислено на ОПС. Доп.тариф.

•В карточку редактирования налоговых данных сотрудника (меню Зарплата - Личные карточки сотрудников, закладка Налоги) добавлены поля для указания тарифа ОПС доп и признаков взятия ОПС доп.

•В карточку зарплаты в расчетном листке добавлены поля указания тарифа ОПС доп и признаков взятия ОПС доп

•В карточку настройки шифра на закладку Налоги добавлен новый параметр Тариф дополнительных взносов на ОПС.

•Добавлена таблица [Тариф ОПС доп] с записями: [1] - п.1 ст.58.3 ФЗ от 24.07.2009 N 212-ФЗ, [2] - п.2 ст.58.3 ФЗ от 24.07.2009 N 212-ФЗ.

•Добавлен новый тип ставки на дату Тариф ОПС доп и записи в таблицу [Ставка по дате] на 2013, 2014, 2015 годы.

•Исправлена форма [Стандартная. Ставки на дату (ред)] - добавлено редактирование новой ставки Тариф ОПС доп.

•Исправлена процедура эталона [Заполнение Ставка по дате] - заполняет таблицу и добавляет ставки для тарифов 1 и 2 с новым типом Тариф ОПС доп.

• Добавлен новый тип субконто сДС по таблице [Тариф ОПС доп], указывается  в расчетном листке страховых взносов в Субконто 2 у докспец ОПС доп, в которой субконто 1=Чел

•Обновлены процедуры эталона, отвечающие за расчет зарплаты и страховых взносов:

-  исправлены процедуры [з/п удержание/Подоходный налог], [з/п расчет одного сотрудника] - заполняют признаки взятия ОПС доп в таблице Зарплата;

-  исправлены процедуры [ЕСН расчет одного сотрудника], [ЕСН Доходы и вычеты с 2010], [ЕСН расчет налога Соцстрах от НС],

-  добавлены функции [Сотрудник.ЗП Тариф ОПС доп], [ЗП.Ставка ОПС доп] и процедуры [ЕСН расчет налога ОПС доп], [ЕСН Затраты по виду дохода ОПС доп], [ЕСН Расчет вычетов ОПС доп]

•Внесены изменения в конфигурацию ''Стандартная_Зарплата_...''

 

[ * ] [19346]

Для карточек редактирования статуса, в которых открыты поля Тип периода, Дата начала, Дата окончания, Количество дней установлено автоматическое заполнение поля Количество дней в зависимости от значений полей Тип периода, Дата начала, Дата окончания и ранее установленного и сохраненного значения Календаря. При добавлении новой записи о статусе сотрудника поле Тип периода заполняется значением из справочника Вид статуса ЗП для этого типа статуса. Если поле Тип периода не заполнено, рассчитывается календарное Количество дней.

 

Разное

 

[ * ] [19277]

Для прикрепленных файлов с расширениями doc, docx, xls, xlsx вызывается при открытии то приложение, которое ассоциировано с их расширением в настройках операционной системы. Это правило распространяется, в том числе и на приложения Openoffice.
