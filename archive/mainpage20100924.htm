Книга покупок и книга продаж

 

[ * ] [17150]

Доработан алгоритм отбора данных и заполнения полей дат в режиме составления Книги покупок. При проставленном признаке Учитывать при составлении дату поступления счет-фактуры в настройке раздела Книги покупок (меню Данные - Настройка - Настройка книги покупок, кнопки Добавить/ Изменить) отбор производится по большей из дат Дата документа или Дата поступления сч-фактуры; в поле Исходная дата включения попадает  значение поля Дата поступления сч-факт, если оно не заполнено, данные не попадают в выборку и в отчет. Если признак Учитывать при составлении дату поступления счет-фактуры не проставлен, то отбор производится по дате документа и в поле Исходная дата включения попадает значение поля Дата счет-фактуры, а если оно не заполнено, то значение поля Дата документа.

 

Сервис

 

[ - ] [17142]

В режиме Проверки целостности и обслуживания базы данных (меню Сервис - Целостность базы данных) убрана кнопка минимизации окна Список ошибок и сообщений. Ранее в случае ее использования возникала патовая ситуация, когда не были доступны для работы ни один режим и ни одно окно.  

 

Информация для разработчиков

 

[ * ] [17149]

Изменена работа следующих дескрипторов запроса, возвращающих запрашиваемый в процессе диалога список значений таблиц Подразделение, Сотрудник, Прочий и ОС, это соответственно:

^DialogPodrIDList^

^DialogKadryIDList^

^DialogProchIDList^

^DialogOsIDList^

Ранее возвращаемое этими дескрипторами значение имело вид: '1,2,3,4,5'. Теперь эти дескрипторы возвращают значение в виде (1,2,3,4,5).

 

Для корректной работы уже имеющихся запросов, использующих эти дескрипторы, сделано следующее:

 

•добавлены новые дескрипторы

^DialogPodrIDListOld^

^DialogKadryIDListOld^

^DialogProchIDListOld^

^DialogOsIDListOld^

работающие по алгоритму '1,2,3,4,5'

 

•при синхронизации БД с релизом до 109 включительно, в запросах, в которых встречаются дескрипторы

^DialogPodrIDList^, ^DialogKadryIDList^, ^DialogProchIDList^, ^DialogOsIDList^,

они заменяются соответственно на дескрипторы

^DialogPodrIDListOld^, ^DialogKadryIDListOld^, ^DialogProchIDListOld^, ^DialogOsIDListOld^

 

Замечание!!!

При использовании указанных дескрипторов не разрешается возврат от релиза 110 к предыдущим версиям программы, иначе требуется ручное переименование этих дескрипторов во всех использующих их запросах.
