Автонумерация

 

[ + ] [19400]

В настройку автонумерации добавлена возможность указания функции для формирования префикса / постфикса. В список  возможных значений для параметров Тип префикса и Тип постфикса добавлено новое значение Функция, при его выставлении становится доступным список значений для нового параметра Функция. В этот список попадают функции, удовлетворяющие следующим ограничениям:

- входящий параметр - дата,

- название функции - не более 10 символов.

В качестве примеров для собственных функций, позволяющих создавать нужные префиксы и постфиксы у номеров, в стандартную конфигурацию BuhtaConfig включено 8 функций, дублирующих уже существующие алгоритмы: [Год_ГГ], [Год_ГГГГ], [Дата_ДД.ММ], [Дата_ММ.ДД], [ДатаДДММГГ], [Мес_ГГ.ММ], [Мес_ММ.ГГ], [Месяц_ММ].

Отчеты

 

[ * ] [19427]

При отправке по почте отчета FastReport 2 с преобразованием в файл формата xls система автоматически преобразует некоторые строковые данные в формат дат или действительных чисел. Для предотвращения подобной ситуации на форму Отправка почтой добавлен новый признак Сохранять все данные как строки (признак становится виден, если для параметра Отправлять в виде отчета установлено значение MS Excel (*.xls)). При его выставлении все данные предаются в символьном формате.

[ + ] [19454]

Добавлено 2 новых события в скрипт ApplicationEvents:

- OnAfterSetTableLock(LockNum, TableName, RecordID),

- OnAfterClearTableLock(LockNum)

События необходимы для того, чтобы пользователь мог на стороне SQLServer организовать доступ к блокировкам записей.

 

- AfterSetTableLock (LockNum: Integer; TableName: String; RecordID: String) - срабатывает, когда процедура TbmTCPClientNet.SetTableLock возвращает номер блокировки записи;

- AfterClearTableLock (LockNum: Integer) - срабатывает при вызове TbmTCPClientNet.ClearTableLock.
