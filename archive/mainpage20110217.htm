Договоры и документы

 

[ * ] [17365]

В настройку видов договора (меню Данные - Настройка - Виды договоров), на закладку Документы добавлен новый параметр Реквизиты счета-фактуры, который отвечает за то, какие  значения заносятся в поля Номер счета-фактуры и Дата счета-фактуры создаваемого документа. Варианты значений:

- Нет - поля Номер счета-фактуры и Дата счета-фактуры документа не заполняются,

- Из договора - поля Номер счета-фактуры и Дата счета-фактуры документа заполняются значениями полей Номер счета-фактуры и Дата счета-фактуры договора,

- Из документа - в поля Номер счета-фактуры и Дата счета-фактуры документа заносятся значения номера и даты документа.

 

Для вновь создаваемых видов договоров значением по умолчанию параметра Реквизиты счета-фактуры является значение Нет.

Для существующих видов договоров в целях совместимости с предыдущими версиями программы действует следующее умолчание при обновлении эталоном 115 релиза и выше: если для указанного вида документа нет автонумерации ни по одному юр.лицу, ни за один период, параметр Реквизиты счета-фактуры заполняется значением Из договора, если имеется хотя бы одна автонумерация - значением Нет.

 

[ * ] [17428]

Доработан интерфейс по вводу шаблонов примечаний в поле Текст карточек договоров и документов. При нажатии на кнопку Т справа от поля ввода Текст, открывается список доступных шаблонов примечаний. Ранее из этого списка выбиралась только одна запись, на которой был установлен курсор. Теперь добавлена возможность пометки строк. При нажатии на кнопку Вставить все отмеченные записи заносятся в поле Текст.

 

Оборотная ведомость

 

[ - ] [17516]

Исправлена ошибка, в результате которой при вызове Карточки редактирования по субконто из Оборотной ведомости не блокировалась  запись, если эта карточка уже была открыта в другом режиме (не в этой оборотной ведомости) текущим или другим пользователем.

 

Безопасность, пароли, доступы

 

[ - ] [17420]

Исправлена обработка прав доступа к полям ввода в стандартной карточке редактирования ТМЦ - в режиме Чтение для справочников ТМЦ и ТМЦ Комплект не был доступен фильтр по периоду на закладке Основной комплект.

 

[ - ] [17441]

При вызове карточек редактирования документов и договоров из пользовательских форм не обрабатывались права доступов к полям таблиц Документ, Договор. Ошибка исправлена.

 

Сервис

 

[ + ] [17320]

В окно управления сервером внесены следующие изменения:

1. Кнопка Запуск БД имеет подменю Запустить все

2. Кнопка Остановить имеет подменю Остановить все

 

При нажатии кнопки Запустить все производится запуск баз, которые находятся в статусе Остановлена, Ожидает остановки, ОШИБКА.

При нажатии кнопки Остановить все производится остановка баз, которые находятся в статусе Работает.

В нижней части окна отображается имя базы данных, над которой производится операция.

 

[ + ] [17485]

В режим синхронизации баз данных, в форму на третьем шага добавлен признак Импорт конфигурации ЗП. Если он выставлен то в базу данных будет произведен импорт файла конфигурации Стандартная_Зарплата_2<две последние цифры номера версии>.mdb (например, для релиза 115 это файл Стандартная_Зарплата_215), который находится в подкаталоге Зарплата того каталога, из которого запускается программа.

 

Внимание!

Файл конфигурации Стандартная_Зарплата_215 содержат только объекты отличные от объектов конфигурации СТАНДАРТНАЯ_ЗАРПЛАТА_ГОД_2010 или не вошедшие в нее. Конфигурация СТАНДАРТНАЯ_ЗАРПЛАТА_ГОД_2010 совпадает с конфигурацией Стандартная_Зарплата_214 от 19.01.2011. Таким образом, импортировать файл Стандартная_Зарплата_215.mdb в режиме синхронизации можно только в том случае, если Вы уверены, что конфигурация СТАНДАРТНАЯ_ЗАРПЛАТА_ГОД_2010 (или Стандартная_Зарплата_214 от 19.01.2011) уже была импортирована в базу.

- [ * ] [17437] - [ЕСН расчет налога с 2010]

- [ * ] [17502] - [З/П Доходы и дни к расчету среднего]

- [ * ] [17495, 17502] - [Заполнение Ставка по дате]
