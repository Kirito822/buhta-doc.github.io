Справочники

 

[ + ] [16865], [16878]

 

•Изменен способ заведения юридического и фактического адресов для Юр.лица (меню Справочники - Настройка - Организации - Юридические лица, карточка редактирования, закладка Адреса), добавлена возможность выбора реквизитов адреса из справочников (при выставленном флаге Использовать справочник ИМНС для заполнения адресов).

•В соответствии со структурными изменениями адресных данных Юр.лица внесены изменения в алгоритм заполнения адресных полей на титульной странице отчетной формы РСВ-1, при этом если в настройках Юр.лица на закладке Адреса заполнен фактический адрес, то он и выводится в отчет, в противном случае распечатывается юридический адрес.

•Исправлены формы [Стандартная. Архивный отчет в эл.виде (Дополнения)], [Стандартная. Архивный отчет в электронном виде] и запрос [Стандартный. Юр.лицо для отчета в электронном виде], отвечающие за выгрузку архивных отчетов в файлы формата XML.

 

Для того чтобы были задействованы указанные изменения, следует: обновить версию, синхронизировать базу данных текущим эталоном и импортировать стандартные конфигурации Стандартная_Зарплата_206 и BuhtaForm.

В соответствии с ПРИКАЗОМ МинФина РФ от 16 декабря 2009 г. N 135н утверждена новая форма Налоговой декларации по налогу на прибыль, в связи с чем добавлен шаблон [Стандартный. Все измененные объекты входят в состав стандартной конфигурации BuhtaForm.mdb

 

[ * ] [16868], [16889]

На форму [Стандартная. Архивный отчет в электронном виде], отвечающую за выгрузку  архивных отчетов в файлы формата XML, добавлено поле Нумеровать файл начиная с № для указания номера, начиная с которого нумеруется файл выгрузки (если такой файл в указанной директории уже существует, ищется следующий незаполненный номер). Если поле Нумеровать файл начиная с № не заполнено, поиск первого незаполненного номера файла с соответствующим названием в указанной директории ведется как и раньше, начиная с 1. Алгоритм работает для всех отчетов, для которых производится выгрузка в файлы формата XML.

 

Исправлена форма [Стандартная. Архивный отчет в эл.виде (Дополнения)] - при выгрузке отчета РСВ-1 (составляется по шаблону [З/П Расчет по взносам на ОПС и ОМС РСВ-1 ПФР]) в файл формата XML значения ИНН и ОГРН указываются так же как и в отчете, с первыми ведущими нулями.

 

Измененные формы [Стандартная. Архивный отчет в электронном виде] и [Стандартная. Архивный отчет в эл.виде (Дополнения)] входят в состав стандартной конфигурации BuhtaForm.

Сервис

 

[ + ] [16618]

Разработан универсальный режим поиска текстовой информации в объектах системы БУХта (меню Сервис, пункт Поиск в объектах).

Поиск осуществляется:

- в текстах форм пользователя (на форме поиска выставлен флаг Формы),

- в текстах запросов (выставлен флаг Запросы),

- в текстах VB-скриптов (выставлен флаг VB скрипты),

- в текстах хранимых процедур, функций, триггеров, view (выставлен флаг Объекты SQL).

Двойное нажатие кнопки мышки в списке найденных объектов открывает соответствующий объект на редактирование, позиционирование производится на строку с первым вхождением искомого текста.

 

[ - ] [16871]

Исправлена ошибка в режимах экспорта /импорта файла конфигурации, в результате которой в базу не добавлялись новые таблицы.

 

Зарплата См. также раздел Конфигурация ''Стандартная_Зарплата_...''

 

[ * ] [16859], [16860]

Внесены следующие изменения в  карточку редактирования налоговых данных сотрудника (Личная карточка сотрудника, закладка Налоги):

•Откорректированы названия полей и добавлены подзаголовки Налог на доходы физических лиц (НДФЛ) и Страховые взносы (С/В). Название Страховые взносы (С/В) меняется на Единый социальный налог (ЕСН), если данные указываются на дату меньшую, чем 01.01.2010 г.

•Добавлены новые параметры: Справка ВТЭК: дата выдачи - дата окончания действия, становятся видимыми при проставленных признаках Назначить/отменить льготу и Льгота инвалидам. Эти параметры не оказывают никакого влияния на расчет С/В, используются только в отчетах (в РСВ-1 -  раздел 4.1 и в Карточке инд.учета страх.взносов). В качестве даты выдачи справки указывается самая ранняя дата по всем налоговым данным сотрудника, в качестве даты окончания действия - самая поздняя.
