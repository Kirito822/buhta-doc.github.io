Сервис

 

[ + ] [19673]

На форму Объекты SQL (режим Сервис - Объекты SQL) добавлена кнопка <Синхронизировать> для вызова пользовательской формы [Стандартная.Синхронизация объектов SQL] (включена в состав конфигурации BuhtaSupport), которая позволяет обновить / добавить в таблицу Объекты SQL записи    о хранимых процедурах, функциях и триггерах, имена которых начинаются с "_" и не начинаются c "_авто_" или "_auto_". При добавлении объекта в таблицу, используется имя "текущей конфигурации" и номер "версии текущей конфигурации" (см. режим Настройка - Общая настройка). При обновлении объекта фиксируется дата изменения объекта.

Модернизация требует синхронизации базы данных эталоном текущей версии, в который добавлена новая хранимая процедура [СинхронизацияОбъектовSQL].

 

[ * ] [19646]

•Модернизирована форма по замене типа субконто [Стандартная. Замена типа субконто] (входит в состав Стандартной конфигурации BuhtaConfig).

- Добавлен контроль полей с именами Тип субконто, Типы субконто.

- Для варианта с заменой типа субконто для выборочных юр.лиц добавлены запросы на подтверждение замены типа субконто для таблиц с отсутствием разделения по юр.лицам.

•Доработан алгоритм создания функции [CheckSubcontoTypeForDelete] - внесены дополнительные проверки на наличие удаляемого типа субконто.

 

[ - ] [17305]

Устранена ошибка в режиме Сервис - Кто работает в сети, возникающая при попытке получить информацию (кнопка <Доп. инфо>) о базе на другом сервере.

 

Отчеты

 

[ * ] [19685]

В структуру файлов конфигурации в таблицы Сводка и Оборотка добавлено поле Колонтитулы, обеспечена передача значения этого поля в режимах импорта / экспорта.

 

[ * ] [19688]

Внесены доработки в настройку Внешнего вида при печати Сводного отчета. При вызове режима Печать списка из контекстного меню Сводного отчета текущий отчет анализируется  на предмет заполненности поля Колонтитулы таблицы Сводка. Если поле Колонтитулы пустое – настройка Заголовок отчета на закладке Внешний вид – доступна, если поле Колонтитулы не пустое - настройка Заголовок отчета на закладке Внешний вид – не видна.

 

[ - ] [19684]

Исправлены следующие ошибки в режиме печати Сводного отчета:

1.Отчет печатался со второго листа, если значения первого поля отчета выходили по ширине за пределы страницы.
2.Нарушалась нумерация страниц отчета, если набор данных полностью умещался на страницу и итоги печатались на новом листе.

 

Прайс-лист

 

[ - ] [19629]

В режиме выборки товара по прайс-листу (меню Данные - Прайс-лист, кнопка <Выбрать товар>) неверно обновлялась стоимость при изменении цены и кол-ва выбранного товара. При увеличении количества выбранного товара двойным нажатием левой кнопки мышки или выбором пункта контекстного меню Добавить одну позицию стоимость изменялась только с учетом количества – т.е. рассчитывалась по старой цене. Ошибка исправлена – при изменении выбранного количества любым способом расчет стоимости производится по обновленной цене.  

 

Кадры

 

[ - ] [19720]

Исправлена ошибка при определении ЗП статуса сотрудника при совпадении даты увольнения и даты нового приема на работу. Ошибка проявлялась в режиме составления Табеля учета рабочего времени, в том случае если в настройках режима Зарплата (меню Зарплата – Настройка – Основная настройка) был выставлен флаг у параметра Считать день увольнения последним рабочим днем. Изменение требует также синхронизации базы данных эталоном текущей версии, который включает следующие обновленные функции:

- [Сотрудник.ЗП статус работника],

- [Сотрудник.ЗП статус тип],

- [Сотрудник.ЗП статус тип Основание],

- [Сотрудник.ЗП статус тип Примечание]
