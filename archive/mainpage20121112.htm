Договоры и документы

 

[ - ] [19030]

Исправлены процедуры эталона [Пересчет спецификаций документа от суммы] и [Пересчет спецификаций договора от суммы] - устранена ошибка деления на ноль при пересчете спецификаций с нулевой суммой и ненулевым количеством.

 

[ * ] [19075]

На форму копирования документов (вызывается из контекстного меню списка документов, пункт Копировать или по горячим клавишам Ctrl+C) добавлен новый признак Снять признак генерации. Этот параметр становится доступным если:

1. признак Копировать спецификации                НЕ ПРОСТАВЛЕН

2. признак Копировать проводки                ПРОСТАВЛЕН

3. признак Выполнить генерацию                НЕ ПРОСТАВЛЕН

Когда флаг Снять признак генерации выставлен, то документ копируется с проводками и без спецификаций.  В новом документе-копии будет снят признак генерации Проведен, а в проводках для поля ГенТип установлено значение 0, что означает возможность их редактирования.

 

Справочники

 

[ + ] [18559]

В базу данных системы Бухта добавлена новая таблица [Кадры Трудовая книжка Стаж] для заполнения сведений из Трудовой книжки сотрудника.  Работа с этой таблицей ведется через кадровую карточку сотрудника (меню Справочники - Сотрудники или Зарплата - Личные карточки сотрудников, кнопка Кадровая карточка).

Пользователю следует после синхронизации базы данных эталоном текущей версии зайти в настройки справочника сотрудников и подключить закладку Кадры Трудовая книжка Стаж. Сотрудник, а затем на этой закладке настроить поля карточки редактирования и запрос для просмотра записей таблицы [Кадры Трудовая книжка Стаж].

Если в указанной таблице имеются записи по текущему сотруднику, то на закладке Главная на панели Стаж становится доступной кнопка <Из Трудовой книжки>, по нажатию на которую годы-месяцы-дни страхового стажа заполняются согласно данным из таблицы [Кадры Трудовая книжка Стаж] по следующей формуле:

Суммарное кол-во дней в периодах с [Даты начала] по [Дату конца] по тем записям, в которых проставлен признак Вкл в страховой стаж, делится на 30 - это суммарное кол-во месяцев. Суммарное кол-во месяцев  делится на 12 -  это кол-во лет. Кол-во месяцев вычисляется как [суммарное кол-во месяцев] минус [кол-во лет *12]. Кол-во дней вычисляется как [суммарное кол-во дней] минус [кол-во лет *12*30] минус [кол-во месяцев * 30].

[ * ] [19100]

В карточках настройки Оборотных ведомостей и Сводных отчетов для Личных отчетов открыто на редактирование поле Тип (Общий/Личный), ранее это поле было доступно для редактирования только в том случае, когда в настройках прав доступа для Оборотной ведомости и Сводных отчетов (соответственно в разделах Отчетность - Оборотная ведомость и Отчетность - Сводный отчет) было установлено значение Полный.

Если для Оборотных ведомостей и Сводных отчетов не установлено полного доступа, то при изменении статуса у этих отчетов с Личного на Общий выдается предупреждающее сообщение о необходимости настройки прав доступа к текущему отчету, так как при отсутствии полного доступа к общим отчетам он станет недоступен.

 

Формы

 

[ + ] [18879], [19091]

Разработана форма [Стандартная. Создание счетов-фактур на аванс] и следующие SQL-процедуры к ней:

- [_СФ на аванс. Зачет авансов пред.периода]

- [_СФ на аванс. Зачет оплаты текущего дня]

- [_СФ на аванс. Зачет возвратов текущего дня]

- [_СФ на аванс. Формирование счетов-фактур на аванс по возврату]

- [_СФ на аванс. Формирование счетов-фактур на аванс]

 

Указанные объекты входят в состав конфигурации BuhtaForm.

 

Форма предназначена для ежедневного отслеживания полученных авансов и их автоматического зачета. Основные условия применения формы - используются 3 бухгалтерских счета для учета Авансов, Оплаты, Реализации. На счете Аванса на 2-м субконто необходимо фиксировать платежные документы.

 

Пользователю следует указать значения параметров, запрашиваемые на форме, в том числе помимо перечисленных выше счетов -  вид документов для зачета аванса, оплаты и возвратов, и вид документа для счета-фактуры на аванс, а также вид спецификаций  для каждого случая.

 

Процедуры выполняются в указанном порядке за каждый день заданного периода:

- Зачет авансов закрывает сальдо Авансов на сальдо Реализации.

- Зачет оплаты текущего дня закрывает кредит Оплаты на сальдо Реализации.

- Зачет возвратов текущего дня  -  это зачет возврата денег с дебета Оплаты на сальдо Аванса (если оно имеется).

- СФ на аванс по возврату – создается счет-фактура, если образовалось кредитовое сальдо Реализации.

- СФ на аванс – создается счет-фактура по кредитовому сальдо Оплаты.

 

По окончанию работы процедур выводится список созданных документов. Пользовательская форма [Стандартная. Список объектов] и запрос

[Документы - результат расчета], которые используются при этом, входят в состав конфигурации BuhtaConfig.
