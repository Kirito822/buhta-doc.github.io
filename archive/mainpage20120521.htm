Доработан отчет Налоговая декларация по налогу на добавленную стоимость (меню Отчеты - Формы налоговой отчетности - Налоговая декларация по НДС):

•Добавлена возможность для заполнения раздела 2. На экран формы [Стандартный. Налоговая декларация по НДС] добавлен признак Заполнять Раздел 2. Если признак проставлен, становится видимой кнопка для указания списка организаций, по которым составляются страницы Раздела 2. После нажатия на кнопку <Составить> открывается отчет, заполненный без Раздела 2 - по старым алгоритмам. Его надо закрыть, он сохраняется в архиве, потом открывается и заполняется Раздел 2 (для каждой из указанных организаций - отдельная страница). После сохранения второго отчета в архиве, первый (без Раздела 2) из него удаляется.

Для Раздела 2 следует настроить новый запрос [Стандартный. Налоговая декларация по НДС Раздел 2]. Особенности его настройки:

- для заполнения строк 060, 080, 090, 100 используются SQL_Aлгоритмы  соответственно 1, 2, 3, 4. Для разнесения сумм по отдельным организациям введена переменная @OrgId, условие с ее использованием пользователь прописывает на закладке SQLWhere карточки настройки алгоритма.

- значение для строки 050 (код ОКАТО) берется из текущего юр.лица

- для строк 040 (КБК) и 070 (Код операции) в запросе предусмотрена заглушка в виде SQL_Строк - настраивается пользователем.

•В разделе 3 исправлен расчет Суммы НДС для строк 070 и 080. Ранее расчет производился по ставке 10% и (или) 18% от значения налоговой базы, сейчас - умножением налоговой базы на коэффициенты 10/110 и (или) 18/118. Изменен шаблон [Стандартный. Налоговая декларация по НДС 2009].

•В запрос [Стандартный. Налоговая декларация по НДС 2009] добавлены SQL-алгоритмы для заполнения в разделе 4 граф 2,3,4,5 для строк 1010401, 1010402.

 

Все измененные и добавленные объекты, связанные с указанной модернизацией, входят в состав конфигурации BuhtaForm.mdb.

[ + ] [18604]

Для рабочего стола Комплекса БУХта разработан новый элемент Папка, для удобства работы с ярлыками АРМ путем их визуального распределения по группам.

Ярлыки АРМ с рабочего стола БУХты можно перемещать в папку методом drag-and-drop. Внутри одной папки можно создавать другие папки.

Создание папки осуществляется вызовом режима Создать папку из контекстного меню рабочего стола или контекстного меню уже имеющейся папки, если мы создаем вложенную папку. При создании папки задаются ее наименование, уникальное в пределах пользователя, и иконка, которые могут быть в дальнейшем изменены (режим Изменить контекстного меню рабочего стола и папки).

Удаление папки с рабочего стола производится вызовом режима Удалить контекстного меню рабочего стола. При этом все вложенные в папку элементы  (ярлыки АРМ и другие вложенные папки) перемещаются на рабочий стол. Содержимое внутренних папок при этом не изменяется. Удалить полностью папку или ярлык АРМ из папки невозможно, можно только переместить их на рабочий стол (пункт контекстного меню Переместить на рабочий стол или "горячая" клавиша Delete).

 

Модернизация требует синхронизации базы эталоном текущего релиза, в результате которой должны создаться таблицы [Папка], [Папка АРМ меню]

и view [Папка view].

 

Зарплата см. также Конфигурация ''Стандартная_Зарплата_...''

В Delphi-скрипт добавлены следующие методы:

- SetSemaforOn - добавляет слева на Главную инструментальную панель кнопку с изображением мигающего фонаря;

В VB-скрипт класса ApplicationEvents добавлено событие OnSemaforClick, которое наступает при нажатии на кнопку с изображением фонаря на инструментальной панели  или рабочем столе.
