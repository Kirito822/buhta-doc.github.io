Договоры и документы

 

[ + ] [20383], [20384], [20387]

Для обеспечения работы с терминалами сбора данных (ТСД) выполнены следующие преобразования:

 

•Изменена структура базы данных (требуется ее обновление  эталоном  текущей версии):

1) добавлены таблицы:

- [ТСД режимы] (поля: Таблица (значения "Договор"/"Документ"), Вид (номер вида договора/документа), Ключ таблицы, Логин),

- [ТСД спец] (поля: ТСД режимы (ключ таблицы [ТСД режимы]), Штрих-код )

2)  в таблицы [Вид договора]/[Вид документа] добавлены поля : Работа с ТСД, ТСД.Поле спец.для записи кода, ТСД.Поле спец.для записи кол-ва.

 

•В структуру файлов  конфигурации добавлены новые поля для таблиц [Вид договора], [Вид документа] и обеспечен экспорт/импорт  их  значений.

 

•В карточки настроек видов договоров/документов на закладку Отображение добавлены новые параметры:

- Работа с ТСД (логическое поле),

- Поле догспец/докспец  для записи ТМЦ (возможные значения Субконто 1-5, Корр.субконто 1-5),

- Поле догспец/докспец  для записи количества (возможные значения Количество, Количество 2, Нетто, Брутто).

 

•Разработан  режим ввода информации с терминала сбора данных в спецификации договора/документа. Форма ввода,  которая   вызывается из карточки редактирования договора/документа по нажатию на кнопку <Режим ТСД>,  отображает информацию об активированном для ввода договоре/документе и списке считываемых штрих-кодов.  По нажатию  на кнопку <Сохранить>  введенная информация записывается в спецификацию  договора/документа в соответствии с настройками вида договора/документа.

Новый запрос [ТСД спец], добавленный в рамках указанной модернизации, включен в конфигурацию BuhtaConfig.

 

Отчеты

 

[ * ] [20369]

Внесены изменения в алгоритм составления Налоговой декларации по налогу на прибыль организаций (меню Отчеты – Формы налоговой отчетность – Налоговая декларация на прибыль):

 

•Если на Листе 02 значение строки 060 <0, то обнуляются значения строк  100, 120, 180, 190, 200.

•Значение для строки 080 Приложения  №2 к листу 02 берется из строки 350 Приложения  №3 к листу 02.

•В Приложении № 4 к листу 02 обнулены расчетные формулы.

•На форму предварительного задания параметров  добавлен новый признак Вычислять долю налоговой базы по ФОТ и остат.стоимость ОС (становится видимым, если проставлен признак Заполнять Приложение 5 по обособленным подразделениям), если он проставлен, то доля налоговой базы по каждому обособл.подразделению считается по ФОТ и по ост.стоимости ОС:

доля налоговой базы рассчитывается как (%ФОТ + %ОС)/2, где

%ФОТ = 100 * ФОТ за период по обособл.подразделению / ФОТ за период по предприятию

%ОС = ((100 * Ост.стоимость ОС на начало периода по обособл.подр./Ост.стоимость ОС на начало периода по предпр.) +  (100 * Ост.стоимость ОС на конец периода по обособл.подр./Ост.стоимость ОС на конец периода по предпр.))/2.

Если признак Вычислять долю налоговой базы по ФОТ и остат.стоимость ОС не проставлен, открывается поле для указания доли налоговой базы, одинаковой для всех обособленных подразделений.

•На форму предварительного задания параметров  добавлен новый признак Данные об авансовых платежах - из архивного отчета за пред.период. Если он проставлен, открывается поле для выбора отчета за предыдущий период, из которого следует брать данные для заполнения авансовых платежей.

На основании  данных за предыдущий период формируются значения следующих строк Листа 02:

В том случае, если отчетный период 1 квартал:

- значение строки 220 равно значению строки 330 из архивного отчета,

- значение строки 230 равно значению строки 340 из архивного отчета.

За  остальные отчетные периоды:

- строка 220 текущей декларации = строке 190 тек. декл. + строка 300 из архивного отчета,

- строка 230 текущей декларации = строке 200 тек. декл. + строка 310 из архивного отчета.

Алгоритм действует начиная с периодов 2014 года.
