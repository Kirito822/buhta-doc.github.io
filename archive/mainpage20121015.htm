Книга покупок и книга продаж

 

[ * ] [18937]

•Внесены изменения в шаблоны [Журнал полученных счетов-фактур 2012] и [Журнал выставленных счетов-фактур 2012] для Журнала учета счетов-фактур (меню Данные - Книга покупок, Книга продаж, кнопка Журнал, кнопка Печать по шаблону):

- для полей Сумма и Суммы НДС установлен формат с двумя знаками после запятой,

- значение поля Сумма теперь заполняется значением переменной Сумма всего (ранее было Сумма спецификации всего).

•В карточку настройки Вида документа (меню Данные - Настройка - Виды документов) на закладку Главная добавлена настройка Код способа выставления счета-фактуры. Значения выбираются из Классификатора вида Код способа выставления СФ.  Значение настройки автоматически указывается в поле Код способа выставления С-Ф при создании документа соответствующего вида. Если значение этой настройки не указано, то при создании документа соответствующего вида в поле Код способа выставления С-Ф устанавливается значение 1 - на бумажном носителе.

 

[ * ] [18939]

Изменен подход к отображению корректировочных счетов-фактур (в дальнейшем корр. СФ) в печатных формах Книг покупок и продаж и Журнала учета С-Ф. Номер и дата самого корр.СФ  заносятся в колонки для корр.СФ, а в колонки для СФ - номер и дата исходного СФ. В строке с исходным СФ информация о корр.СФ не заполняется.

 

Введено понятие корр.СФ фиксирующих уменьшение стоимости. Для указания таких СФ в таблицу Документ добавлено новое поле - признак Уменьшение стоимости. Этот признак можно заполнить на этапе создания корректировочного документа с помощью формы [Стандартная. Корректировка счета- фактуры] – в форму добавлен новый параметр Проставить в корректировочном счете-фактуре признак "Уменьшение стоимости".

Для корр.СФ, у которых проставлен признак Уменьшение стоимости, изменен алгоритм заполнения Журнала. Такие СФ включаются в "противоположный журнал"  -  СФ из  Книги покупок включаются в журнал учета выставленных СФ (часть 1) и не включаются  в журнал полученных СФ (часть 2). СФ из Книги продаж включаются в часть 2 Журнала и не включаются в часть 1.

Для того чтобы определить в какой конкретно раздел журнала должны попадать такие СФ, в настройки Книг покупок и продаж добавлена новая настройка Раздел журнала выставленных / полученных С-Ф для включения корр.С-Ф на уменьшение стоимости. При совпадении имен разделов в Книгах покупок и продаж наименование раздела можно не указывать.

Новое поле Уменьшение стоимости добавлено также в таблицы Журналов учета СФ, т.к. при пересоставлении Книг покупок, продаж эти записи удаляются отдельно от основного списка.

Включение корр.СФ в Книги покупок или продаж регулируется настройкой проводок. В релизе 124 был рекомендован план настройки генерации проводок для  создания проводки с общей суммой коррекции СФ. В настоящем релизе следует добавить настройки  генерации проводки от уже имеющихся проводок, которые в соответствии с добавленными в них фильтрами для положительной и отрицательной суммы должны формировать проводку - для Книги продаж (сумма проводки >0) или для Книги покупок (сумма проводки <0).

 

Измененные объекты в рамках указанной модификации входят в состав конфигурации BuhtaConfig.

В связи с добавлением новых полей требуется также обновление структуры базы эталоном текущей версии.

Информация для разработчиков

 

[ + ] [18948]

Добавлена возможность при отправке отчетов по электронной почте из режима предварительного просмотра задавать имя прикрепляемого файла через переменную MailDefault, которая может быть определена в VB-скрипте одного из запросов, по которому составляется комплект отчетов, входящих в файл. Например:

 

Sub ViewOnGetValue(Column, ByRef Value) ''

SetGlobalVar "MailDefault", "text"

End Sub

 

Если переменная не заполнена, то в качестве имени файла указывается значение "Отчеты", которое может быть отредактировано вручную.

Удаление переменной MailDefault происходит при закрытии окна предварительного просмотра отчетов.

 

[ + ] [18959], [18981]

В окно редактора скрипта Dream добавлено два новых режима - вызываются из меню Исходный код или по нажатию кнопки на инструментальной панели:

- Завернуть SQL-выражение в TQuery, TStringList или String - При вызове этого режима вызывается окно, в котором  следует заполнить запрашиваемые параметры, после чего по нажатию на кнопку <Сохранить> составленный программный код добавляется в текст скрипта по месту расположения курсора.

- Извлечь SQL из выдел.текста и записать в буфер обмена - Режим позволяет из выделенного текста скрипта извлечь фрагменты заключенные в одинарные кавычки и сохранить их в буфере обмена.
