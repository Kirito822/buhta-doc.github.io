Разное

 

[ * ] [19202]

Внимание!!!

В связи с тем, что при работе с временными таблицами не поддерживается преемственность SQL Server 2012 с предыдущими версиями SQL Server,  внесены многочисленные изменения в программный код модуля, в процедуры эталона и во все конфигурации комплекса БУХта. Этот релиз требует обязательного обновления структуры базы и импорта всех используемых конфигураций.  

 

Книга покупок и книга продаж

 

[ + ] [19178]

В экранные формы Книги покупок и Книги продаж добавлена колонка Номер и дата корректир. счета-факт. продавца.

 

[ * ] [19217]

Обновлены формы Варианты печати книги покупок, Варианты печати книги продаж (вызываются из режимов Книга покупок (продаж), кнопка <Составить отчет>):

- С форм удалены варианты отчетов Форма 2003 года, Форма 2004 года

- Пункт Форма 2006 года переименован, новое название Книга покупок  (Книга продаж)

- Пункт Дополнительные листы переименован, новое название Дополнительные листы книги покупок (книги продаж).

 

Книга доходов и расходов

 

[ * ] [19229]

Внесены следующие изменения в режимы составления и печати Книги доходов и расходов:

•В меню Данные - Книги доходов и расходов добавлено два новых пункта:

- Данные предыдущих периодов - при выборе этого пункта вызывается пользовательская форма [Книга доходов и расходов. Справка и убытки],

- Настройка УСН - при выборе этого пункта вызывается пользовательская форма [Настройка УСН].

Для указанных пунктов меню можно назначить вызов форм с другим названием, см. режим Настройка - Инструменты - Привязка меню к формам.

 

•При попытке печати отчетов при незаполненных настройках УСН  выдается предупреждающее сообщение.

•Исправлены ошибки округления в отчете [Книга доходов и расходов (раздел I) 2011].

 

Изменения требуют синхронизации базы эталоном текущего релиза (исправлены процедуры эталона [Контроль корректности информации] и [Заполнение привязки меню к формам]) и импорта конфигурации BuhtaConfig (группа Книга доходов).

Справочники

 

[ * ] [19206]

В справочнике ТМЦ длина поля Штрих-код увеличена до 50 символов. Требуется синхронизация базы эталоном текущей версии.

 

Списки

 

[ * ] [19193]

В режиме выбора обработчика для строк списка (горячие клавиши Alt_F12 или пункт контекстного меню Пользовательские формы) изменен принцип отбора пользовательских форм. Теперь для справочников, имеющих разделение по видам, включая также списки спецификаций в карточках редактирования документов и договоров, при выборе обработчика отображаются только те формы, что относятся к записи, на которой стоит курсор, если нет отмеченных записей. Если включен режим пометки строки, то показываются только те пользовательские формы, которые относятся к  пересечению множеств форм соответствующих видам отмеченных записей.
