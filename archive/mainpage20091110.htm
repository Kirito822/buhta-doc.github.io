ВНИМАНИЕ !!!

 

Начиная с релиза 100 изменилась нумерация версии у объектов конфигурации. Теперь в номере версии перед номером релиза стоит цифра 2. Так объекты 100-го релиза имеют номер версии 2.100.

Прайс-лист

 

[ * ] [16400], [16408]

Модернизирован режим Прайс-лист (вызывается из меню Данные):

 

При установленном в настройках Прайса параметре Использовать упаковки, в Прайс-листе в режиме выборки товара появляется колонка для ввода количества, в которую заносится Количество упаковок.

Для того чтобы иметь возможность использовать эту колонку для ввода Количества2 (веса),  в процедуры PriceFormShow и PriceListEditOnChange для соответствующих событий добавлена глобальная переменная Прайс.Использовать количество 2, для которой в VB-скрипте следует установить значение Истина. Кроме того, желательно изменить заголовок колонки присвоением нового значения UpakColumn.Caption (например, UpakColumn.Caption = "Вес").

Поскольку в этом режиме пересчет количества в вес и обратно не производится автоматически, алгоритм пересчета следует прописать в скрипте по событию OnSelectingTMCKol. Для определения, в какую из колонок производился ввод, нужно проверить параметр Param("Изменена колонка") - он содержит заголовок измененной колонки.

Колонка  формы Выбранный товар (вызывается по нажатию на кнопку Выбранный товар в окне Прайс-листа), в которой отображалось количество упаковок, теперь содержит  или Количество упаковок или Количество 2 - в зависимости от того, что содержится в соответствующей колонке в окне Прайс-листа.

Кадры

 

[ * ] [16380]

Изменен алгоритм редактирования кода в Табеле учета рабочего времени (меню Кадры - Табель учета рабочего времени).

Если в ячейке указываем код табеля, у которого не проставлен признак Заполнять часы в табеле (устанавливается в режиме Кадры - Настройка - Коды табеля), то количество часов, указанных под редактируемой ячейкой, обнуляется.

Если указываем код табеля, у которого проставлен признак Заполнять часы в табеле, а часы для соответствующей ячейки табеля не указаны, то количество часов проставляется из ЗП Статуса сотрудника, если в ЗП Статусе указан неполный рабочий день, или 8, если полный рабочий день.

 

Настройка

 

[ + ] [16010]

В общую настройку программы (меню Настройка - Общая настройка, закладка Общие настройки) добавлен новый параметр Использовать стандартный список для выбора вида при добавлении записи.

Если у этого параметра выставлен флаг, то при добавлении новой записи в справочник, имеющий разделение по видам, вызывается стандартный список видов этого справочника. Список видов отображается в иерархическом виде, если для него указана соответствующая настройка.

Если флаг у указанного параметра не выставлен, то при добавлении новой записи в справочник вызывается, как и в прежних версиях, встроенная форма Выберите вид <название справочника> с линейным списком видов справочника.

Эта же форма вызывается, если добавление записи в справочник осуществляется через АРМ, действие Добавить запись.  Поскольку в данном режиме доступна фильтрация данных по видам,  и как следствие невозможно отображение дерева иерархии, настройка Использовать стандартный список для выбора вида при добавлении записи в этом случае не учитывается.

[ + ] [16414]

Добавлен новый алгоритм для ежемесячных начислений Готовая сумма Период=Месяц ЗП.

Если в ЗП Инфо указана дата начала и она меньше первого дня расчетного месяца ЗП, то в расчетном листке в соответствующем шифре в поле Дата начала проставляется первый день расчетного месяца, если в ЗП Инфо указана дата конца и она больше последнего дня расчетного месяца ЗП, то в расчетном листке в соответствующем шифре в поле Дата конца проставляется последний день расчетного месяца.

Используется для таких начислений, как например Отпуск по уходу за ребенком до полутора лет, когда в оперативной информации проставляется весь период, а в расчетных листках требуется указание дат расчетного месяца.

Алгоритм будет добавлен в базу при ее синхронизации эталоном текущей версии.
