Договоры и документы

 

[ + ] [20681]

Режим Создание документа из договора: добавлена возможность вызова пользовательского режима взамен стандартного  или же отключения стандартного режима с помощью скрипта.

Для этого в VB-Script карточки договора добавлено событие OnUserCreateDoc с параметрами: SkipMode, SkipResult. Параметры могут принимать логическое значение true или false. Если в событии установить параметр SkipMode=True, то не производится стандартный вызов карточки создания документа из договора. Параметр SkipResult устанавливается в true в случае, если пользовательский режим создал документ.

Формы

 

[ + ] [20675]

Разработан механизм отображения подсказок пользователю по стандартным режимам. В общую настройку программы (меню Настройка – Общая настройка) на закладку Внешний вид добавлен параметр Форма для отображения подсказки пользователю, для которого нужно указать - выбрать из списка имя формы помощи (чтобы форма попала в список выборки в ее названии должна присутствовать строка "Помощь").

Если для параметра указано значение, то в системном меню форм отображается пункт Помощь, при инициализации которого вызывается указанная в настройке форма помощи. В ней доступен ряд параметров, описывающий режим из которого вызвана форма.  В качестве примера в конфигурацию BuhtaSupport включена форма [Стандартная. Помощь по формам].

 

Настройка

 

[ + ] [20676]

Добавлен функционал для подключения сканера к компьютеру без заполнения таблицы Подключение оборудования. Изменен также интерфейс на главной закладке формы Настройка справочника 'Оборудование' (меню Настройка – Оборудование - Общая настройка):

- Добавлено поле Сканер ШК, ссылка на таблицу Оборудование.

- Добавлен подзаголовок Виды оборудования для прежних полей формы.

Теперь при посылке сообщения WM_CHAR префикс/постфикс сначала проверяется через таблицу Подключение оборудования. Если оборудование с такими настройками  не найдено, ищется в общей настройке, если найдено - генерируется событие OnBarCode.

Модернизация требует синхронизации БД текущим эталоном -  изменен view [Подключенное оборудование view].

 

Сервис

 

[ + ] [20671]

В режиме рассылки сообщений добавлена возможность подключения пользовательской формы выбора адресатов.  Для этого в общую настройку программы (меню Настройка – Общая настройка) на закладку Интернет/Почта/Сообщения добавлен флажок Подключить пользовательскую форму для выбора адресата сообщения и поле для выбора этой формы. Для того, чтобы форма попала в список выбора, ее имя должно содержать строку "сообщен" и любую из строк "адресат" или "получател". При установленном флажке выбор адресатов осуществляется с помощью указанной формы, при сброшенном все работает как раньше.

В качестве примера создана форма [Стандартная. Выбор адресатов сообщения], которая входит в конфигурацию BuhtaSupport.

3.        Макс.база для взносов в ПФР= 670000 руб.,

расчет страховых взносов  теперь изменен таким образом, что предельная величина для каждого взноса берется своя

4.        Макс.ежемес.страх.выплата ФСС = 65330 руб.,

5.        МРОТ= 5965 руб.,

6.        Мин.пособие по уходу = 2718.34 руб.,

7.        Мин.пособие по уходу за 2реб=5436.67 руб.

 

•Так как с 2015 года ставка НДФЛ с дивидендов увеличилась до 13%  в таблицу Ставки на дату добавлена ставка на  01.01.2015  с типом Налог с дивидендов =13.

 

Указанная модернизация будет произведена при  синхронизации БД эталоном текущей версии, в котором обновлены следующие процедуры:

- [dbo].[Заполнение Ставка по дате],

- [dbo].[ЕСН расчет налога Соцстрах от НС],

2. с иностранцев ВЖНР с 01.01.2015 снять Вид дохода ЕСН =9 При такой настройке для ФСС берется ставка 2.9%.
