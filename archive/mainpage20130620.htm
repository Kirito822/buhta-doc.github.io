Сервис

 

[ + ] [19566], [19613], [19623]

Добавлен новый стандартный тип субконто Дат; таблица, на которой основан этот тип, не указывается. Обновление доступно после синхронизации базы данных эталоном текущей версии.

 

ВНИМАНИЕ:

Для тех клиентов, у которых уже имеется тип субконто Дат, основанный на пользовательской таблице, в обязательном порядке  требуется его переименовать ПЕРЕД установкой 143 –го или более позднего релиза.

Рекомендуемая последовательность действий при переименовании пользовательского типа субконто Дат:

- В версии, более ранней чем 143, следует добавить новый тип субконто, например ДТ, с указанием той же самой таблицы, что и в типе субконто Дат.

- Перезагрузить программу.

- Импортировать форму [Стандартная. Замена типа субконто] из файла конфигурации BuhtaConfig 143 релиза.

- Произвести замену типов субконто с помощью формы [Стандартная. Замена типа субконто], указав в качестве старого типа субконто значение Дат, в качестве нового - значение ДТ, замену производить для всех юр.лиц, с пересозданием SQL-процедур "Генерация проводок из документов".

- Удалить тип субконто Дат.

- Синхронизировать базу данных эталоном 143 релиза.

 

[ * ] [19509]

Исправлены формы [Стандартная. Настройка закладок], [Стандартная. Настройка закладок. Безопасность, пароли, доступы

 

[ * ] [19520]

В режимах создания, изменения и копирования учетной записи пользователя, группы и роли  введен запрет на логины, названия групп и ролей с начальными и конечными пробелами.

 

[ * ] [19521]

В форму копирования учетной записи пользователя добавлен новый параметр Копировать список юр.лиц. Если этот признак выставлен, то при копировании учетной записи пользователя производится копирование списка допустимых юр.лиц для этого пользователя (список вызывается по нажатию на кнопку Юр.лица на закладке Учетные записи). Параметр Копировать список юр.лиц доступен, если в базе более одного юр.лица и отмечена хотя бы одна запись в списке юр.лиц.

 

[ - ] [19522]

В режиме копировании учетной записи пользователя при выставленном признаке Копировать привязку АРМ привязка к АРМ-у, совпадающему по имени с именем логина той учетной записи, с которой производится копия, теперь НЕ копируется.

 

[ * ] [19570]

В настройке прав доступов оптимизированы режимы сохранения внесенных изменений для следующих окон:

- Состав группы (закладка Группы, кнопка Состав),

- Состав роли (закладка Роли, кнопка Состав),

- Принадлежность к группам (закладка Учетные записи, кнопка В группах),

- Принадлежность к ролям (закладка Группы, кнопка В ролях).

 

[ - ] [19593]

[ - ] [19626]
