Прайс-лист

 

[ * ] [19753]

Модифицирован режим работы с Прайс-листом (меню Данные - Прайс-лист):

1.Добавлена информационная строка в шапку формы, доступна в скрипте прайса как ClientLabel.
2.В режим настройки Прайс-листа (меню Данные - Настройка - Прайс-лист) добавлен вариант включения поиска или набора количества По типу колонки (настройка прайса Нажатие цифр).

В этом варианте:

- при нажатии цифр, если курсор стоит на колонке ввода, то вводится количество, если на других колонках - включается поиск;

- при нажатии Enter в колонках "не для ввода" курсор переходит в поле ввода количества;

- при нажатии F4 в колонке ввода открывается форма ввода количества.

3.    Добавлен доступ к событиям перемещения по списку ТМЦ в Прайсе - OnClick, OnChangeNode, что позволяет вывести в информационную строку параметры текущего ТМЦ, в том числе информацию о набранном количестве и скидках.

 

Модернизация требует обновления базы данных эталоном текущей версии.

 

Книга доходов и расходов

 

[ * ] [19788]

Внесены изменения в права доступа для Книги учета доходов и расходов:

- В разделе Данные - Книга учета доходов и расходов настройки прав доступов (меню Настройка - Пользователи / Пароли / Доступы) из названий доступов убрана подстрока "(раздел I)". Новые названия теперь: Создание книги учета, Изменение книги учета, Просмотр книги учета.

- Действие настроек в разделе Данные - Книга учета доходов и расходов распространено на пункты меню Данные предыдущих периодов и Настройка УСН. А так как для этих пунктов задействованы еще и права доступа к привязанным формам (см. Релиз 144 Заявка [19585]), то  результирующим для пунктов меню Данные предыдущих периодов и Настройка УСН будет пересечение прав доступов в разделе Данные - Книги учета доходов и расходов и в разделе Сервис - Формы пользователя.

 

Модернизация требует обновления базы данных эталоном текущей версии и импорта измененных форм [Книга доходов и расходов. Справка и убытки.], [Настройка УСН] из стандартной конфигурации BuhtaConfig.

 

Генерация проводок

 

[ + ] [19774]

Добавлена возможность в процедурах генерации проводок наравне со стандартным набором сортировок использовать пользовательскую функцию. Функция должна иметь следующие параметры: Тип субконто, Субконто, Ключ документа, Ключ докспец, и возвращать строку, участвующую в сортировке FIFO. Длина названия функции должна быть не больше 34 символов. При выполнении этих условий название функции появляется в списке возможных значений  для параметра Сортировка в режиме настройки генерации проводок (меню Настройка - Генерация проводок). Подробнее см. раздел Примеры функций. Функции сортировки FIFO.

 

Списки

 

[ * ] [19772]

При выгрузке в Excel из режима Оборотной ведомости и стандартных списков добавлена обработка имени файла выгрузки, создаваемого по умолчанию - угловые скобки заменяются на круглые, все остальные символы, недопустимые в имени файла, пропускаются.

 

Зарплата см. также Конфигурация ''Стандартная_Зарплата_...''

 

[ * ] [19771]

Откорректирована кадровая карточка сотрудников. Для полей Образование и Послевуз. образование (закладка Образование) назначены раздельные списки для выбора возможных значений (ранее для этих полей список был единым). Модификация требует синхронизации базы эталоном текущей версии.

 

[ - ] [19806]

Изменена форма [Стандартная. Архивный отчет в эл.виде (Дополнения)] (конфигурации BuhtaForm) - в выгрузке отчета, составленного по шаблону [З/П Расчет по взносам на ОСС 4-ФСС 2013], исправлена выгрузка значений ОКВЭД и номера корректировки.

 

[ * ] [19809]

Изменена функция эталона [dbo].[ЗП.Ставка подоходного], теперь если в функции прямо не указана ставка для кода дохода, то она берется из таблицы [Вид дохода].
