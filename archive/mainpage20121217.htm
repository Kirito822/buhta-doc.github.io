Отчеты

 

[ * ] [19119]

В соответствии с приказом Росстата от 24 июля 2012 г. N 407 внесены следующие изменения в алгоритмы формирования отчета Сведения о численности и заработной плате работников (Форма № П-4) :

 

•Модернизирован принцип сбора данных для отчета - произведена детализация по ОКВЭД:

a) Изменения в эталоне:

- в таблицу Подразделение добавлено поле [ОКВЭД],

- исправлена процедура [dbo].[Контроль корректности информации],

- добавлена процедура [dbo].[Заполнение ОКВЭД]

После синхронизации базы данных эталоном текущей версии  в таблицу Подразделение будет добавлено новое поле [ОКВЭД], в справочник Классификатор добавлен новый вид ОКВЭД и произведено заполнение Классификатора данного вида.

b) Изменения в версии - в карточку редактирования подразделения добавлено поле для указания ОКВЭД. По нажатию на кнопку [...] рядом с полем на экране появляется список ОКВЭД (все записи справочника Классификатор вида ОКВЭД). При выборе строки полю присваивается значение поля Номер таблицы Классификатор.

 

•В конфигурацию Стандартная_Зарплата_237 добавлены новые форма [З/П Печать Сведений о численности-ЗП П-4. 2013г] и шаблон [Сведения о численности-ЗП П-4.Форма 2013г], по которым строится отчет. Подробнее см. описание Конфигурация ''Стандартная_Зарплата_...''

 

[ * ] [19131]

Сведения о фин.сост. организации П-3]  - начиная с января 2013 года отчет Сведения о финансовом состоянии организации (Форма № П-3) печатается по новому шаблону [Стандартный. Сведения о фин.сост. орг-ции П-3 2013].

 

[ * ] [19145]

Исправлена форма [Настройка форм отчетности. Из режима печати отчета], которая используется в отчетах с настраиваемыми показателями.

Теперь при сохранении запроса, если имеется хотя бы один показатель, настроенный на Дебетовое сальдо вх, Дебетовое сальдо исх, Кредитовое сальдо вх или Кредитовое сальдо исх (поле [Тип сальдо]), то в секции SQL-Before запроса вызывается процедура [_Сводные обороты по субконто], в остальных случаях для типов Свернутое сальдо вх, Свернутое сальдо исх, Дебет, Кредит вызывается процедура [_Сводные обороты по субконто_1], которая формирует активное и пассивное сальдо по Субконто 1, игнорируя остальные уровни субконто.

При этом, как и раньше, процедура [_Сводные обороты по субконто_1] (или [_Сводные обороты по субконто]) вызывается только если хотя бы у одного показателя заполнено поле [Список субсчетов], иначе вызывается процедура [Сводные обороты].

 

Изменения связаны с необходимостью оптимизации вычислительного процесса.

[ - ] [19138]

Внесены следующие исправления в печатные формы Книг покупок и продаж:

- Для корректировочных счетов-фактур в столбце Номер и дата корректировочного счета-фактуры продавца указывалась дата не корректировочной, а исходной с-ф. - ошибка исправлена.

Модификация требует синхронизации базы эталоном текущей версии (обновлены процедуры [Книга покупок показать результат] и [Книга продаж показать результат]).

 

Настройка

- DOC

- XLS

- DOCX

- XLSX

- JPG

- PDF

- *.*

Значение по умолчанию для самого параметра Расширение по умолчанию для списка документов  - DOC.

Информация для разработчиков

 

[ + ] [19101]

В стандартную форму выбора значения (TbmLookupForm)  добавлено событие проверки корректности выбранной записи - с возможностью запрета закрытия формы.

 

Событие формы называется OnLookupFormValidate.

Параметр DontCloseLookupFormBecauseOfValidate определяет возможность закрытия формы.

 

Пример скрипта:

 

Sub FormOnLookupFormValidate(ByRef DontCloseLookupFormBecauseOfValidate) ''

 gr = FindComponent("Grid")

 gr.View.RecNo=gr.FocusedNode.Data

 DontCloseLookupFormBecauseOfValidate = gr.View.RecordID>50

 if DontCloseLookupFormBecauseOfValidate then

   bmInformation "Выберите запись с ключом меньше 50"

 end if

End Sub

 

Синхронизация

 

[ * ] [19141]

В эталон текущей версии добавлена процедура Синхронизация.Выполнение SQL скриптов пользователя

Эта процедура запускается в конце процедуры синхронизации, что позволяет пользователю помимо выполнения скриптов корректировать еще и схему данных.

В лог файл выводится следующая информация:

******* Запуск процедуры выполнения скриптов пользователя *******

Запуск процедуры выполнения скриптов пользователя - выполнен.
