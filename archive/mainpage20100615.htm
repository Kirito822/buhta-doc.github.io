Отчеты

 

[ + ] [16776]

Модернизирован отчет Акт сверки взаиморасчетов (меню Отчеты - Расчеты с дебиторами и кредиторами - Акт сверки взаиморасчетов или через пользовательскую форму [Стандартная. Акт сверки взаиморасчетов 2010]) - добавлена возможность составлять отчет в выбранной валюте по выбранному курсу.

 

В том случае, если на форме задания параметров для параметра Отчет составлять в указано значение валюте открывается  поле Валюта отчета.

Если значение в поле Валюта проводок не совпадает со значением в поле Валюта отчета, открываются поля:

- Курс (Коммерческий / ЦБ),

- меню на дату операции / на фиксированную дату,

- Дата курса (при выбранном значении на фиксированную дату).

 

При составлении отчета, если выбран вариант Отчет составлять в валюте и валюта проводок не совпадает с валютой отчета, выполняется пересчет всех сумм в валюту отчета по заданному курсу (выбирается из справочника Курсы валют - см. меню Данные) на дату операции или ту, что указана в поле Дата курса. Если выбран вариант пересчета На дату операции, остаток на начало периода пересчитывается по курсу на дату начала периода, а остаток на конец периода рассчитывается как сумма пересчитанных значений остатка на начало периода и итоговой суммы за период.

Если указан признак Показывать суммы в валюте проводки, второй строкой в отчете выводится сумма проводки без  пересчета.

Формы

 

[ + ] [16925]

Добавлена пользовательская форма [Стандартная. Замена округления в SQL_Алгоритмах] для пакетной замены в SQL_Алгоритмах входящих в запрос значения параметра Сумма в (параметр отвечает за округление данных выводимых в отчет).

На экране формы следует указать значение поля Сумма в, подлежащее замене, новое значение этого поля и список запросов, в которых надо произвести замены. По нажатию кнопки Список запросов для замены открывается окно со списком запросов, в настройках алгоритмов которых присутствует значение параметра Сумма в, указанное на форме как подлежащее замене. Следует отметить нужные запросы и нажать на кнопку Выбрать - окно со списком запросов будет закрыто, на форме отобразится перечень выбранных запросов. Замена производится по нажатию на кнопку Выполнить.

 

Форма [Стандартная. Замена округления в SQL_Алгоритмах] входит в состав стандартной конфигурации BuhtaForm текущей версии.

 

Зарплата См. также раздел Конфигурация ''Стандартная_Зарплата_...''

 

[ + ] [16892]

Для режима Зарплата - Свод по зарплате добавлена возможность составления отчетов по пользовательским шаблонам.

При выставленном на форме Свод начислений и удержаний по з/п признаке По шаблону открывается поле со списком шаблонов:

- [Стандартный. Свод начислений и удержаний по З/П] - аналог встроенного шаблона, без запросов, печатается по данным из встроенного свода;

- [Стандартный. Свод по З/П по шифрам и сотрудникам] с запросами [Стандартный. Свод по З/П Шифры] и [Стандартный. Свод по З/П Сотрудник] (со скриптом и формой [Стандартная. Свод по З/П Сортировка]) - аналог встроенного шаблона, но печатается по новым таблицам  [#Свод ЗП Сотрудник] и [#Свод ЗП Шифр] без настроек вcтроенного шаблона: ориентации листа, размещения на листе и т.д.;

- [Стандартный. Свод по З/П по шифрам ИТОГО] с запросом [Стандартный. Свод по З/П по шифрам] - отражает cписок начислений и удержаний без разбиения по сотрудникам.

 

Шаблоны и запросы доступны для пользовательского редактирования.
