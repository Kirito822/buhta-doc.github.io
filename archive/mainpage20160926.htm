[ * ] [21740]

В таблицу Подразделение добавлено новое поле ОКПО (30 символов) -  используется для обособленных подразделений. Модифицирована карточка редактирования подразделения. Пользователю после смены релиза и обновления структуры БД эталоном текущей версии следует добавить новое поле в карточку редактирования и у обособленных подразделений проставить в этом поле соответствующие идентификационные номера.

[ * ] [21828]

Внесены следующие изменения в статистическую форму № П-1 СВЕДЕНИЯ О ПРОИЗВОДСТВЕ И ОТГРУЗКЕ ТОВАРОВ И УСЛУГ (все измененные и добавленные объекты входят в стандартную конфигурацию BuhtaForm):

•В соответствии с Приказом Росстата от 04.08.2016 N 388 Приложение № 3 к Форме № П-1 утрачивает силу с 2017 года. Поэтому начиная с 2017 года, отчет составляется в новом формате без Приложения №3. Исключена также строка 28 в разделе 3.

•Изменен алгоритм составления Формы № П-1 с учетом наличия обособленных подразделений.

При наличии у юридического лица обособленных подразделений отчет должен быть составлен как по каждому обособленному подразделению, так и по юридическому лицу без этих обособленных подразделений. В связи с чем на экран формы для составления отчета [Стандартный. Сведения о производст. и отгрузке П-1] добавлен признак Отдельные отчеты для обособл. подразделений (по ОКПО). Если признак проставлен, то вместо одного отчета создается несколько. При этом группировка данных в один отчет производится по коду ОКПО обособленных подразделений (в карточке Подразделения должен быть проставлен признак Обособленное подразделение и заполнено поле ОКПО). Отдельные отчеты создаются для всех обособленных подразделений с одинаковым кодом ОКПО, который печатается на титульном листе и в примечании к названию архивного отчета. Данные по всем не обособленным подразделениям также попадают в отдельный отчет с кодом ОКПО текущего юр.лица.

Перед первичным составлением отчета при выставленном признаке Отдельные отчеты для обособл. подразделений (по ОКПО) следует через форму настройки запросов (вызывается нажатием командной кнопки с изображением треугольника справа от кнопки <Составить отчет П-1>) настроить следующие запросы:

- за текущий период - запрос [Стандартный. Форма П-1/1 по подразделениям],

- за предыдущий месяц – запрос [Стандартный. Форма П-1/2 по подразделениям],

- за месяц прошлого года, соответствующий текущему – запрос [Стандартный. Форма П-1/3 по подразделениям].

Для облегчения этой задачи в форму настройки запросов добавлена кнопка <Копировать из другого запроса>, по нажатию на которую  можно выбрать форму и запрос, показатели из которого надо скопировать в текущий запрос. В данном случае следует выбрать для формы значение П1, для запросов - значения:

- за текущий период  - запрос [Стандартный. Форма П-1/1],

- за предыдущий месяц – запрос [Стандартный. Форма П-1/2],

- за месяц прошлого года, соответствующий текущему – запрос [Стандартный. Форма П-1/3]

Затем требуется перенастроить скопированные показатели для работы с учетом обособленных подразделений. В частности должен быть перенастроен показатель [s2_21_B], который отвечает за код по ОКВЭД.

Также следует откорректировать запрос [Стандартный. Форма П-1/ОКЕИ по подразделениям], отвечающий за Раздел 5 отчета. Запрос, входящий в конфигурацию BuhtaForm, дается в качестве примера. Пользователю необходимо самостоятельно настроить значения полей: [s5_0_А], [s5_0_Б], [s5_0_В], [s5_0_Г], [s5_0_Д], [s5_0_1], [s5_0_2], [s5_0_3], [s5_0_4], [s5_0_5], и для параметра запроса Сальдо/Оборот выбрать из предлагаемого списка и установить значение [Стандартный. Форма П-1/ОКЕИ].
