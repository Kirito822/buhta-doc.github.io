Расчеты

 

[ * ] [18501]

Откорректирована рассылка сообщений о начале/окончании работы в режиме Закрытия (открытия) периода и установки новой оперативной даты (меню Расчеты - Закрытие (открытие) периода). [ * ] [18475], [18510]

Внесены следующие изменения и дополнения в интерфейс и алгоритмы расчеты страховых взносов:

 

•Теперь взносы по шифрам в расчетных листках ЗП с кодом тарифа 9 Доходы иностранных граждан пишутся в отдельную докспец. Если код тарифа 9 Доходы иностранных граждан, то взносы считаются по настройкам в ЗП Налоги в карточке сотрудника - если для взноса проставлено Полный, то взнос рассчитывается, если Нет - то не рассчитывается. При этом ОПС независимо от возраста попадает только в страховую часть. Тарифы взносов для иностранцев берутся по настройке Пониженные тарифы страх.взносов в Основной настройке ЗП.

Исправлены следующие процедуры эталона:

- [ЕСН расчет налога с 2012],

- [ЕСН Затраты по виду дохода с 2010],

- [ЕСН Заполнение шкалы].

•В карточку настройки шифра (меню Зарплата - Настройка - Шифры ЗП) для параметра Тариф по виду дохода добавлено новое значение вид дохода 9 - Доходы иностранных граждан. В случае указанной настройки при расчете заработной платы при виде доходов 3-Материальная помощь, 4-Медикаменты, 5-Натуральные выплаты этот тариф проставляется в расчетном листке ЗП.

•Исправлена процедура эталона [з/п удержание/Подоходный налог] - при расчете ЗП поле Тариф СВ заполняется по настройке шифра или из вида дохода ЕСН, указанного в карточке ЗП Налоги (в прежних версиях не выполнялось для вида дохода 9 Доходы иностранных граждан).

•Исправлена карточка настройки ЗП Налоги (закладка в Личной карточке сотрудника). Теперь при выборе значения в поле Вид дохода проставляются актуальные настройки для взятия СВ, а именно:

- для всех видов дохода  взносы в ПФР = НЕТ, взносы ТФОМС = НЕТ (начиная с 2012г),

- виды доходов 1-5: ФФОМС,ФСС,ОПС = ПОЛНЫЙ,

- вид дохода 6 По договору подряда: ФСС=НЕТ, ФФОМС,ОПС = ПОЛНЫЙ,

- вид дохода 7 Вмененный доход: ФФОМС, ФСС, ОПС = ПОЛНЫЙ,

- вид дохода 8 Упрощенная система налогообложения: ОПС=Полный,  ФФОМС,ФСС =НЕТ,

- вид дохода 9 Доходы иностранных граждан: ОПС=Полный, ФФОМС,ФСС =НЕТ,

- вид дохода 10-11-12: ФФОМС,ФСС,ОПС = НЕТ,

- вид дохода 13 Доходы инвалидов: ФФОМС, ФСС, ОПС = ПОЛНЫЙ.

•Изменена процедура эталона [Заполнение Ставка по дате] - добавлена ставка Округление соцстрах от НС (определяет округление взносов на соцстрах от НС в расч.листке СВ)

Безопасность, пароли, доступы

 

[ + ] [18467]

Добавлена возможность назначения для конкретного пользователя списка юр.лиц, для которых может быть осуществлен вход в систему БУХта.

Интерфейс реализован через систему пользовательских доступов, поэтому вместе с обновлением версии и синхронизацией базы данных требуется импорт следующих объектов, входящих в конфигурацию BuhtaConfig.mdb:

- форма [Стандартная. Логин Юрлицо],

- пользовательский доступ [Логин Юрлицо].

Для того чтобы назначить определенному пользователю список юр.лиц, который он будет видеть при входе в систему БУХта, следует в форме Пользователи /Пароли /Доступы (меню Настройка - Пользователи /Пароли /Доступы) на закладке Учетные записи установить курсор на требуемую запись в списке пользователей и нажать на кнопку Юр.лица. Если у текущего пользователя (того, под которым ведется работа в программе) есть право на пользовательский доступ Логин Юрлицо (раздел в правах доступа Сервис - Пользовательские доступы), то  откроется список юридических лиц. Галочками следует отметить те, которые должны отражаться в списке юр.лиц при входе в БУХту под именем выбранного пользователя.

Если не выделено ни одно юр.лицо, то при входе в систему БУХта отображаться будет весь список. Существует ограничение - невозможно отменить видимость текущего юр.лица для текущего пользователя, т.е. если для текущего пользователя указано хотя бы одно юр.лицо, то текущее юр.лицо тоже должно быть указано.

 

Сервис

 

[ + ] [18452]

В контекстное меню стандартных списков добавлен вызов режима сканирования (меню Сервис, пункт Сканировать или горячие клавиши Alt+S). Полученное изображение сохраняется в виде привязанного файла формата gif.

Режим сканирования для всех стандартных списков кроме таблиц договоров и документов доступен, если у пользователя есть права на добавление привязанных файлов (доступ Сервис-Привязанные файлы). Для вызова режима сканирования из списка документов требуется наличие права на работу с файлами для соответствующего вида документов в разделе доступов Данные - Документы /Виды документов. Для вызова режима сканирования из списка договоров - наличие права на работу с файлами для соответствующего вида договоров в разделе доступов Данные - Договоры /Виды договоров.

 

Метод вызова режима сканирования доступен из DelphiScript VBScript

 

1. Пример вызова из Dream

procedure Proc;

begin

 buhtaGrid1.ScanAndAddImage(); // - использование скобок не обязательно

end;

где buhtaGrid1: TbuhtaGrid;

 

2. Вызов в VBScript аналогичен, использование скобок обязательно

 FilesGrid.ScanAndAddImage();

[ + ] [18468],

Добавлена возможность отправлять сообщение (событие) выбранному пользователю (или всем) в указанные базы, формат сообщения текстовый.

Для этого создана сервисная таблица [Сообщения в BuhtaServer] с полями [База], [Пользователь], [Сообщение], в которую помещаются записи с новыми событиями .

BuhtaServer сканирует таблицу на предмет новых событий и отправляет их пользователям БУХты.

Отправка сообщения осуществляется из базы данных, в которой заполняется таблица [Сообщения в BuhtaServer]. При этом для отправки сообщения необходимо, чтобы хотя бы один пользователь работал в программе БУХта.

При отправке сообщения на клиентскую часть сообщение удаляется из таблицы [Сообщения в BuhtaServer].

При этом доставка сообщения не проверяется, сообщение удаляется в любом случае.

 

Модернизация требует синхронизации БД эталоном текущей версии.
