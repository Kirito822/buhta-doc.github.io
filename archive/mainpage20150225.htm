Документы

 

[ * ] [20720]

Модернизирован режим создания корректировочного счета-фактуры. На пользовательскую форму [Стандартная. Корректировка счета-фактуры]) (входит в состав конфигурации BuhtaConfig), из  которой вызывается указанный  режим, добавлен новый параметр Вид спецификации брать из документа-источника. Если этот параметр  выставлен, то в корректировочном документе спецификации создаются того же вида, что и в исходном документе. При несоответствии видов спецификаций в документе-источнике и документе-результате, т.е. при отсутствии нужного  вида спецификаций для документа того вида, который указан как вид корректировочного документа, спецификации с ошибочным видом скопированы не будут, о чем будет выведено предупреждающее сообщение.

 

Договоры

 

[ * ] [20731], [20737]

В режиме создания доверенности по договору (вызывается из пользовательской формы [Стандартная. Создание доверенности по договору]) модифицирован алгоритм копирования спецификаций договора в спецификации доверенности :

- Добавлено копирование значений для новых полей [Количество 2] и [Ед.изм. 2] в спецификации доверенности. При частичном копировании значение поля [Количество 2] рассчитывается пропорционально введенному новому значению в поле [Количество].

- Введен запрет на частичное копирование при объединении спецификаций

- Для частичного копирования добавлен дополнительный параметр Только с ненулевым количеством или суммой. Если этот параметр выставлен, не производится копирования для  спецификаций с нулевыми значениями в полях Количество и Сумма.

 

Измененные в рамках этой модернизации объекты - форма [Стандартная. Создание доверенности по договору] и процедура [Договор_Доверенность_создание] входят в состав конфигурации BuhtaConfig.

Модернизация требует синхронизации базы данных эталоном текущей версии - в таблицу [Довспец] добавлены два новых поля [Количество 2] и [Ед.изм. 2].

Описание работы с пользовательской формой [Стандартная. Декларация по НДС 2015. Экспорт в XML].

 

Форма [Стандартная. Декларация по НДС 2015. Экспорт в XML], как любая пользовательская форма, вызывается из меню Настройка – Инструменты –Формы пользователя. Если же пользователь не собирается составлять Налоговую декларацию по НДС за периоды, предшествующие 2015, то можно настроить вызов формы [Стандартная. Декларация по НДС 2015. Экспорт в XML] через главное меню Отчеты -  Формы налоговой отчетности - Налоговая декларация по НДС. Для этого следует воспользоваться режимом Главного меню Настройка – Инструменты – Привязка меню к формам.

 

Форма [Стандартная. Декларация по НДС 2015. Экспорт в XML] состоит из нескольких закладок, количество которых определяется настройкой параметров на главной закладке Выгрузка.

Выгружаемые сведения разбиты на следующие группы, каждой из которой соответствует свой параметр, от значения которого зависит, осуществляется ли выгрузка этой информации или нет:

Разделы 1-7 – сведения выгружаются из архивного отчета, составленного по шаблону [Стандартный. Налоговая декларация по НДС 2015]. Если актуального отчета в архиве нет, его следует составить по кнопке <Новый отчет>. Если нужный отчет уже есть в архиве, его следует указать в поле Архивный отчет выбором из предоставляемого списка (после составления нового отчета ссылка на него проставляется в это поле автоматически). При этом в файл XML, соответствующий разделам 1-7, выгружаются названия файлов по разделам 8-12, поэтому надо либо одновременно создавать и их тоже, либо указывать названия ранее созданных файлов в соответствующих полях, которые становятся доступны для редактирования при выставленном параметре Из файла.

Раздел  8 – сведения выгружаются из книги покупок (файл выгрузки с префиксом NO_NDS.8) и из дополнительных листов книги покупок (файл выгрузки с префиксом NO_NDS.81). Если установлены параметры для этих выгрузок, на форме появляется дополнительная закладка Книга покупок, на которой можно просмотреть данные, подлежащие выгрузке.

Раздел  9 – сведения выгружаются из книги продаж (файл выгрузки с префиксом NO_NDS.9) и из дополнительных листов книги продаж (файл выгрузки с префиксом NO_NDS.91). Если установлены параметры для этих выгрузок, на форме появляется дополнительная закладка Книга продаж, на которой можно просмотреть данные, подлежащие выгрузке.

Раздел 10 – сведения выгружаются из журнала учета выставленных счетов-фактур. Если установлены параметры для этой выгрузки, на форме появляется дополнительная закладка Журнал выставл. с/ф, на которой можно просмотреть данные, подлежащие выгрузке. Файл этой выгрузки имеет префикс NO_NDS.10.

Раздел 11 – сведения выгружаются из журнала учета полученных счетов-фактур. Если установлены параметры для этой выгрузки, на форме появляется дополнительная закладка Журнал получ. с/ф, на которой можно просмотреть данные, подлежащие выгрузке. Файл этой выгрузки имеет префикс NO_ NDS.11.

Внимание!

Дополнительные листы Книг продаж и покупок надо выгружать только при выгрузке Уточненной Декларации – должен быть выставлен соответствующий параметр на форме, при этом надо также указать Номер корректировки и Признак актуальности ранее представленных сведений из Книг и Журналов. Если сведения актуальны, на форме следует выставить параметр АКТУАЛЬНЫ ранее представленные сведения из Книг и Журналов, в этом случае сведения из Книг и Журналов не будут попадать в соответствующие файлы.

 

Действия необходимые для обновления функционала по текущей заявке:

1.Обновить версию программы
2.Синхронизировать базу данных эталоном текущей версии.
3.Импортировать новые и обновленные объекты из файлов конфигурации BuhtaConfig и BuhtaForm.
4.Произвести настройку запроса [Стандартный. Налоговая декларация по НДС 2015]
5.В настройке разделов Книг покупок и продаж, данные по которым  подлежат  выгрузке  в файлы XML, следует выставить флаг у параметра Включать в декларацию по НДС.

[ + ] [20739]

Для предварительной проверки данных контрагентов по информации из Единого Государственного Реестра Налогоплательщиков  (ЕГРН) разработаны следующие пользовательские формы:

·        [Проверка контрагента из справочника организаций] - для вызова из списка организаций,

·        [Проверка контрагента из справочника сотрудников] - для вызова из списка сотрудников,

·        [Проверка контрагента из книги покупок] - для вызова из книги покупок,

·        [Проверка контрагента из книги продаж] - для вызова из книги продаж.

В указанных списках следует отметить необходимые для проверки записи и нажать <Shift+Ctrl+M> (для справочника организаций и сотрудников вызов осуществляется также из контекстного меню Пользовательские режимы). В появившемся окне в случае необходимости следует указать дату, в пределах которой действовал контрагент, и нажать кнопку <Выполнить проверку>. Для подробного ознакомления с работой web-сервиса "Проверка контрагента" (FNSNDSCAWS) обратитесь по  ссылке http://npchk.nalog.ru.

Для справочников Организаций и Сотрудников результаты проверки будут записаны в поля:

- _СтатусКонтрагента - ссылка на пользовательский справочник _Проверка контрагента.Статус,

- _СтатусКонтрагентаНаДату - дата последней проверки.

Для книг покупок и продаж результаты проверки отражаются в протоколе, который появляется по завершении  режима  проверки.

Все объекты, добавленные в рамках выполнения заявки, входят в Стандартную конфигурацию BuhtaForm.
