Отчеты

 

[ * ] [17082]

Модернизирована форма составления Налоговой декларации по ЕНВД - [Стандартный. Налоговая декларация по ЕНВД].

На каждой закладке формы добавлен признак Расчет физических показателей. При установленном признаке открываются  кнопка <Расчет показателей> и поле с выпадающим списком для выбора имени запроса (в список выводятся запросы, имя которых начинается с "ЕНВД. Расчет физ.показателя").

При выполнении расчета запрос выполняется за каждый месяц расчетного квартала (указывается в поле Налоговый период); результат запроса размещается в поле соответствующего показателя.

Исправленная форма [Стандартный. Налоговая декларация по ЕНВД] и пример запроса [ЕНВД. Расчет физ.показателя (Сальдо по ОС)] по таблице Сальдо/Оборот входят в состав стандартной конфигурации BuhtaForm.

 

[ * ] [17112], [17124]

В Карточки движения по субконто и по счету (меню Отчеты - Карточка движения по субконто, Отчеты - Карточка движения по счету) добавлен флаг Показывать сальдо и обороты, свернутые в 0. Если он выставлен, то при составлении движения, помимо итоговых сумм по указанному субконто или счету, проверяется также и количество записей по субконто/счету. Если записи есть, то выводится строка со счетом с возможностью ее детализации.

 

[ * ] [17110]

Исправлены формы [Стандартная. Архивный отчет в электронном виде], [Стандартная. Архивный отчет в эл.виде (Дополнения)], отвечающие за выгрузку архивных отчетов:

- в окне для загрузки файла добавлена возможность поиска,

- откорректирован формат выгружаемых данных.

Исправленные формы входят в состав стандартной конфигурации BuhtaForm.

 

Оборотная ведомость

Кадры

В связи с тем, что число штатных единиц при неполной ставке может быть не целым числом, изменен тип данных поля [Количество штатных единиц] для таблицы [Кадры штатное расписание]. Теперь это поле может содержать действительные числа.

 

Генерация проводок

 

[ + ] [16931], [17115]

Модернизирован режим Настройка - Генерация проводок:

 

•Добавлены два новых алгоритма: Списание суммы Кр по FIFO и Списание суммы Дб по FIFO. В отличие от остальных алгоритмов списания по FIFO поиск и контроль позиций сальдо в этом случае выполняется не по количеству, а по сумме. Для суммы Кр выполняется поиск позиций с отрицательной суммой, для суммы ДБ - с положительной - в пределах Докспец.Суммы. Если  не хватает сальдо для покрытия суммы докспец, то производится генерация проводки на оставшуюся сумму без заполнения субконто из сальдо. В проводку записывается ошибка генерации, в заголовке карточки редактирования документа прописывается сообщение об ошибке.

 

• В алгоритмы списания по FIFO добавлена возможность дополнительной обработки ошибок FIFO. В процедуру генерации добавлена переменная @FIFOErrorMode. Если этой переменной в настройке генерации присвоить значение 1 (карточка редактирования для генерации, кнопка <+SQL>, на закладке +SQLAfter прописать Set @FIFOErrorMode=1), то сообщение об "ошибке" меняется на сообщение о "предупреждении". Если установить значение 2 (Set @FIFOErrorMode=2), то проводка с ошибкой не создается, в заголовке карточки редактирования документа отсутствует сообщение об ошибке/предупреждении.

 

• Добавлены следующие варианты сортировки для позиций сальдо:

- по Документам (Дата, номер) по возрастанию и убыванию:

+Документ.Дата,+Документ.Номер

-Документ.Дата,-Документ.Номер

- по Партиям (Срок реализации, номер) по возрастанию и убыванию:

+Партия.[Срок реализации],+Партия.Номер

-Партия.[Срок реализации],-Партия.Номер
