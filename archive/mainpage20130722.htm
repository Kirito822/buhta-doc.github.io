Безопасность, пароли, доступы

 

[ * ] [19585]

Добавлена обработка доступов к режимам Данные – Книга доходов и расходов – Данные предыдущих периодов и Данные – Книга доходов и расходов – Настройка УСН. Пункты меню становятся доступны, если в режиме Настройка - Пользователи / Пароли / Доступы есть доступ к пользовательским формам, указанным в настройке Привязка меню к формам (режим Настройка-Инструменты- Привязка меню к формам). Если для этих пунктов меню формы не указаны, то для режима Данные предыдущих периодов проверяется доступ к форме Книга доходов и расходов. Справки и убытки, для режима Настройка УСН - к форме Настройка УСН.

 

Настройка

 

[ + ] [19355]

Добавлена поддержка протокола SSL для рассылки почтовых сообщений с вложениями отчетов:

•В настройку параметров почты и в Общие настройки системы (закладка Соединение с Интернет) добавлен флаг SSL. Отправка почты при установленном параметре производится с использованием протокола SSL.

•Добавлена возможность сохранения  параметров е-mail в Локальных настройках пользователя (меню Настройка - Локальные настройки, закладка Параметры e-mail). - Состав и функциональность этих параметров соответствует параметрам Соединения с Интернет из Общих настроек системы, которые начиная с этой версии, используются в качестве образца и для совместимости с предыдущими версиями системы.

- Для переноса значений из Общих настоек в Локальные на закладку с локальными настройками добавлена кнопка <Взять параметры из общей настройки>.

- Заполнение параметров почты по умолчанию при рассылке почтовых сообщений производится из Локальных настроек пользователя, в случае если они отсутствуют или не заполнены - из Общих настроек.

- Если в окне рассылки почтовых сообщений выставлен флаг Сохранить настройки, то при закрытии этого окна, измененные значения параметров почты, если таковые изменения производились, будут сохранены в Локальных настройках, в отличие от прежних версий, в которых изменения сохранялись в Общих настройках.

[ - ] [19654]

•При запуске клиентской части под Windows 7 или выше, каждое подключение с одного и того же компьютера под одним и тем же логином использовало отдельную лицензию, тем самым нарушалось правило - с одного компьютера с одним и тем же логином на одну лицензию разрешается два подключения. Ошибка исправлена.

•Внесено дополнительное изменение: запуск под виртуальной машиной считается работой под терминальным доступом, а следовательно каждое подключение использует отдельную лицензию.

 

Информация для разработчиков

 

[ + ] [19565]

В VBScript прайс-листа добавлена переменная (только для чтения) VBPriceMode.

Переменная может принимать одно из значений:

VBPriceMode = pmPrimary - прайс-лист открыт в обычном режиме;

VBPriceMode = pmInsertToDog - прайс-лист открыт из карточки договора, пользователь нажал кнопку Добавить позиции из прайса;

VBPriceMode = pmCorrectPrice - прайс-лист открыт из карточки договора, пользователь нажал кнопку Корректировать спецификации по прайсу.
