Зарплата См. также раздел Стандартная_Зарплата

 

[ * ] [21661]

Исправлена процедура эталона [Импорт справочника Кодов налогоплательщика ЕСН и ОПС] - добавлен новый вид Классификатора Код по месту нахождения НДФЛ и в Классификатор внесены значения этого вида.

 

[ * ] [21631], [21687]

Изменена форма [Стандартная. Составление платежных ведомостей] (входит в стандартную конфигурацию BuhtaConfig):

1. Внесены изменения в алгоритм создания спецификации документа: Субконто 4 заполняется типом субконто Мес и заносится значение из поля на экране Расчетный месяц - месяц, на выплату доходов которого составляется ведомость.

2. На форму добавлен новый признак Создать спецификацию с суммой НДФЛ. Если он выставлен, то появляются поля для выбора вида спецификации и Шифра ЗП, и при составлении платежного документа в него помимо основной спецификации добавляется спецификация по НДФЛ указанного вида по указанному шифру ЗП.

3. Добавлена возможность для составления ведомости за один день в рамках расчетного месяца (признак За один день месяца).

 

[ * ] [21694]

В таблицу Ставка по дате (меню Зарплата – Настройка – Ставки на дату) добавлен новый размер МРОТ равный 7500 рублей с 01.07.2016. Модернизация требует обновления БД эталоном текущей версии – внесены изменения в процедуру эталона [Заполнение Ставка по дате].

[ - ] [21718]

В режимах Сводный отчет, Карточка движения по субконто, Карточка движения по счету при печати иерархического списка по отмеченным записям (режим Печать списка из контекстного меню) сбивалась иерархия. Ошибка исправлена.

 

Списки

 

[ * ] [21702]

Проведена доработка функции проверки существования записи. Теперь наличие записи в таблице проверяется не только в таблице, но и в представлении для этой таблицы. Распознается ситуация, когда запись присутствует в представлении таблицы и отсутствует в самой таблице. В этом случае карточка редактирования открывается в режиме чтения.

Для того чтобы таблицы Архив и Файл отвечали указанной модернизации, созданы представления [Архив view] и [Файл view], и в схеме данных для этих таблиц указаны имена их представлений.

Модернизация требует обновления БД эталоном текущей версии.

 

Сервис

 

[ * ] [21704]

В реестр добавлен новый параметр настройки подключения базы. Теперь последний вошедший пользователь сохраняется в двух местах:

1. Как и раньше в строке Parameters, оставлено для совместимости

2. В отдельном параметре LastUser. Этот параметр является приоритетным. Т.е. если он сохранен в реестре, то именно значение параметра LastUser подставляется в карточку логина, иначе выбирается из строки Parameters.
