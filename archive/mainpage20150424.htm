Отчеты

 

[ + ] [20736]

В связи с тем, что начиная с 2015 года  предприятиям СПБ нужно платить налог на имущество и рассчитывать авансовые платежи исходя не только из оценочной, но и из кадастровой стоимости объекта по категории недвижимого имущества, внесены следующие изменения в расчет налога на имущество:

•В таблицу ОС добавлены 3 поля:

 - _Брать налог на имущество,

 - _Кадастровый номер,

 - _Кадастровая стоимость

Расчет налога на имущество теперь производится ТОЛЬКО по тем ОС, у которых выставлен признак Брать налог на имущество. Перед использованием нового алгоритма расчета, для пакетной установки этого признака следует воспользоваться пользовательской формой [ОС. Установка признака Брать налог на имущество], которая вызывается из списка ОС выбором обработчика или через контекстное меню.

Для объектов с заполненным полем Кадастровая стоимость выполняется расчет налога исходя из этого показателя. Налоговая декларация и Налоговые расчеты по авансовым платежам в этом случае дополняются разделом для ОС, имеющих кадастровую стоимость.  Если ОС, стоимость которого рассчитывается по кадастровой стоимости, представляет собой помещение (не здание!), в поле [_Кадастровый номер] следует вводить через косую черту ('/') два номера - номер здания и номер помещения (пример: 78:01:0000001:1234/78:01:0000001:55555).

•Изменена форма для составления декларации по налогу на имущество и отчета по авансовым платежам.

Теперь она называется [Стандартная. Декларация по налогу на имущ. 2015]. Для того чтобы подключить  вызов этой формы к пункту  Главного меню Отчеты – Формы налоговой отчетности – Налоговая декларация по налогу на имущество следует воспользоваться режимом Настройка – Инструменты – Привязка меню к формам.

В зависимости от указанного периода составляется Налоговая декларация (если период – год) или Налоговый расчет по авансовым платежам (в остальных случаях). Данные по авансовым платежам в отличии от предыдущих версий, где для этой цели использовался специальный запрос,  заносятся в таблицу _Показатели из архивных отчетов, откуда выбираются при построении отчета за следующий период.

Форма для составления декларации дополнена интерфейсом для выгрузки данных отчета в файлы XML. Экспорт декларации  в дальнейшем будет производиться только из этой формы (не из режима Архив составленных отчетов !). Выгрузка в XML возможна только при установленном на форме флажке В архив. Выгрузка данных производится из того отчета, который указан в поле Архивный отчет (по умолчанию это последний составленный отчет), в случае если это поле не заполнено, следует выбрать нужный отчет из предоставляемого по нажатию на кнопку […] списка. В поле Путь для выгрузки в XML требуется указать каталог для выгружаемых файлов.

Модернизация требует обновления БД эталоном текущей версии и импорта конфигурации BuhtaForm.

[ * ] [20864]

Внесены следующие изменения в форму [Стандартная. Декларация по НДС 2015. Экспорт в XML] (входит в состав конфигурации BuhtaForm).

- В качестве постфикса к названию выгружаемых файлов добавляется  не номер п/п, как ранее, а уникальный идентификатор.

- В файлах выгрузки Книг и Журналов  не создается тэг <СвПокуп> (<СвПрод>), если ИНН - пустая строка.

- В файлах выгрузки добавлена проверка на ошибочно введенные начальные и конечные пробелы у показателей ИНН и КПП.

- Исправлены названия тэгов платежного документа в Доп.листе Книги покупок вместо ДокПдтвОпл,  НомДокПдтвОпл, ДатаДокПдтвОпл  теперь используется ДокПдтвУпл, НомДокПдтвУпл, ДатаДокПдтвУпл.

- Суммовые показатели выгружаются в формате с двумя знаками после запятой.

[ * ] [20844]

Модифицирована карточка настройки платежных поручений типа Перечисление по запросу - добавлена новая возможность выбора поля платежного поручения, которое заполняется указанным значением группировки из запроса. В частности это позволяет составлять платежные поручения для обособленных подразделений с группировкой по коду ОКТМО с указанием значения кода. Для этого следует в запрос, задействованный в настройке поручения добавить поле ОКТМО (значение соответствующего поля из карточки обособленного подразделения - поставщика расчетного листка). По этому полю запроса сделать группировку, проставить новый признак Указать в поле ПП значение группировки и выбрать поле платежного поручения, в которое следует класть значение группировки, а именно ПП Код ОКАТО.

Модернизация требует:

- синхронизации БД эталоном текущей версии - в таблицу [ЗП перечисление] добавлены новые поля настройки [ПП Запрос Поле группировки Ok] и [ПП Запрос Поле группировки],

- обновления пользовательской формы [Стандартная. Составление платежных поручений] - входит в состав конфигурации BuhtaConfig.
