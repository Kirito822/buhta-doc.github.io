[ - ] [21166]

В пакетном режиме перегенерации проводок Расчеты – Перегенерация проводок в документах не всегда правильно отрабатывал фильтр по таблице Документ.

При выставленном параметре Удалять все проводки перед генерацией в случае применения фильтра по документу производилась перегенерация ВСЕХ проводок удовлетворяющих фильтру по периоду и виду документа, а не только тех, что входили в фильтр по документу.

Если в процессе генерации участвовала таблица #FIFOSaldo (при любом значении параметра Удалять все проводки перед генерацией) удалялись проводки по фильтру Период и Вид документа, которые не вошли в фильтр по документу.

Ошибки исправлены.

 

Отчеты

 

[ * ] [21170]

Разработан режим создания ответа на Требование ИМНС о представлении пояснений по Налоговой декларации по НДС.

На форме [Стандартная. Декларация по НДС 2015. Экспорт в XML]:

- добавлены кнопка <Добавить Ответ на Требование> и поля для указания номера и даты Требования (кнопка и поля становятся видимыми, если текущими являются закладки со списками записей Книг или Журналов);

- добавлена закладка Ответ на Требование о пояснениях.

 

Алгоритм работы.

 

При нажатии на кнопку <Добавить Ответ на Требование> в таблицу [_ДеклНДС_Пояснение] добавляются записи с типом Подтверждение с указанными реквизитами Требования (номер и дата) по всем отмеченным записям списка текущей закладки (Книги покупок-продаж или  Журнала СФ). Тип Подтверждение означает, что Сведения, в отношении которых направлено Требование, соответствуют первичным учетным документам и не допущены ошибки заполнения декларации. В случае, если в Сведениях, в отношении которых направлено Требование, выявлены ошибки заполнения декларации, при этом первичные учетные документы не содержат указанных ошибок, надо присвоить этому сведению тип Расхождение и для каждого такого сведения добавить строку с типом Пояснение, в которой заполняются только корректируемые значения исходной записи.

Перевод сведения из Подтверждения в Расхождение и добавление соответствующего сведения Пояснение делается автоматически по всем отмеченным сведениям при нажатии на кнопку <Перевести в Расхождение> на закладке Ответ на Требование о пояснениях, на которую выведен список записей новой таблицы [_ДеклНДС_Пояснение]. Этот список обновляется по нажатию:

1) кнопки <Выборка> - выводятся все записи,

2) кнопки <Показать ответ на Требование> - выводятся записи с указанным номером.

Выгрузка пояснений в файл производится по нажатию кнопки <Выгрузить> на закладке Ответ на Требование о пояснениях.

Само Требование загружается на этой же закладке по нажатию кнопки <Загрузить из XML-файла с Требованием>. При поиске на имя файла с Требованием наложено условие "1165050*.xml".

 

Добавленные и исправленные в рамках текущей модернизации объекты:

- форма [Стандартная. Декларация по НДС 2015. Экспорт в XML],

- таблица [_ДеклНДС_Пояснение],

- запрос [_ДеклНДС_Пояснение для XML]

входят в состав конфигурации BuhtaForm.

Настройка

 

[ - ] [21175]

В режиме Настройка – Инструменты – Запросы не работал импорт запроса, привязанного к таблице Сальдо/Оборот. Ошибка исправлена.

Импорт строки Сальдо/Оборота связанной с запросом в этом режиме производится по следующему алгоритму:

•Значение поля Название импортируемой строки не совпадает ни с одним значением этого поля в таблице Оборот в БД  - добавляется новая строка в таблицу Оборот, импортируемый запрос ссылается на нее.

•В БД в таблице Оборот есть строка с таким же названием, что и у импортируемой строки, совпадают также все значения полей группы Конфигурация - новая строка Сальдо/Оборота в таблицу Оборот не добавляется, импортируемый запрос добавляется и ссылается на строку Сальдо/Оборота, уже имеющуюся в БД.

•В БД в таблице Оборот есть строка с таким же названием, что и у импортируемой строки, но значения полей группы Конфигурация другие - новая строка Сальдо/Оборота в таблицу Оборот не добавляется, импортируемый запрос добавляется в БД, но без какой-либо ссылки на Сальдо/Оборот.
