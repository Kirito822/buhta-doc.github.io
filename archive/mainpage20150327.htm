Книга покупок и книга продаж

 

[ + ] [20819]

Изменен способ хранения показателя Код вида операции СФ, который используется при работе с Книгами покупок и продаж. Ранее этот показатель указывался и считывался из Документа, привязанного к строке Книги покупок/продаж, теперь он хранится среди показателей самих Книг.  Для Книги покупок добавлена также дополнительная обработка показателя Код вида сделки.

В связи с вышеуказанным модифицированы следующие алгоритмы и пользовательский интерфейс в режимах работы с Книгами покупок и продаж:

•В экранные формы Книги покупок и Книги продаж добавлено поле Код вида опер.

•В карточки редактирования строк Книги покупок и Книги продаж, а также Журналов учета счетов-фактур добавлено поле Код вида операции – ссылка на Классификатор.

•Модифицированы настройки Книги покупок и Книги продаж – в карточки редактирования разделов Книги покупок добавлены поля Код вида операции, Код вида сделки, в карточки редактирования разделов Книги продаж добавлено поле Код вида операции – ссылки на Классификатор.

•Учтены значения новых полей в режимах копирования строк в Книгах покупок/продаж и при копировании строк из Книг в Журналы.

•В режиме составления книг настройка на Код вида операции и Код вида сделки в разделах Книг имеет приоритет перед  значением поля Код вида операции СФ из Документа.

•При печати Журнала выставленных и  Журнала полученных  счетов-фактур в случае отсутствия в записи журнала значения для Кода вида операции в отчетах указывается значение соответствующего поля из настройки раздела, а в случае и его отсутствия значение поля Код вида операции СФ из  Документа.

•При печати Журнала полученных счетов-фактур в случае отсутствия в записи журнала значения для Кода вида сделки в отчетах указывается значение соответствующего поля из настройки раздела.

•Доработана форма [Стандартная. Декларация по НДС 2015. Экспорт в XML] (входит в состав конфигурации BuhtaForm) – которая используется для выгрузки декларации по НДС в файлы формата XML. При обработке данных из Книг покупок / продаж и Журналов учета счетов-фактур добавлена выборка значений для Кода вида операции и Кода вида сделки (закладки на форме по Книгам и Журналам) и  выгрузка значений этих параметров в файлы XML.

Модернизация требует синхронизации  БД эталоном текущей версии и импорта  конфигураций BuhtaConfig и BuhtaForm.

 

Отчеты

 

[ * ] [20760]

В соответствии с Приказом ФНС России от 26.11.2014 N ММВ-7-3/600@ добавлен новый вариант Декларации по налогу на прибыль. Вызов отчета осуществляется через обновленную пользовательскую форму [Стандартный. Налоговая декл. по налогу на прибыль].

Поскольку согласно Письму ФНС России от 05.02.2015 N ГД-4-3/1696@ Декларацию  по налогу на прибыль за 2014 г. можно подать и по старому, и по новому шаблону, на форму [Стандартный. Налоговая декл. по налогу на прибыль] добавлен признак Форма 2013 г., который виден, если отчет составляется за период  2014 год.

Декларация печатается по прежним шаблонам для периодов по 3 квартал 2014 года, а также за 2014, если признак Форма 2013 г. выставлен.  Если же отчет составляется за отчетный период 2014 г, когда признак Форма 2013 г. снят, или же за периоды более поздние, чем 2014 г., вызывается новый шаблон [Стандартный. Нал. декл.по налогу на прибыль 2014], который использует новый запрос [Стандартный. Налоговая декларация по НалПриб 2014]. Показатели, входящие в этот запрос, относятся к Листу 3 и Приложениям Листа 2 декларации.  Они отличны от прежнего шаблона и пользователю следует настроить их самостоятельно в запросе [Стандартный. Налоговая декларация по НалПриб 2014].

Исправлена пользовательская форма [Стандартная. Архивный отчет в электронном виде], отвечающая за выгрузку отчета в файлы формата XML – добавлена выгрузка отчета, составленного по шаблону [Стандартный. Нал. декл.по налогу на прибыль 2014]. Для этого отчета страницы 3, 11,12,13, 14.1…14.N,15,18,19 выгружаются, только если проставлен номер в поле Стр. Данные со страницы 20 и далее не выгружаются.

Объекты измененные и добавленные в рамках этой модернизации входят в файл конфигурации BuhtaForm.

Автонумерация

 

[ - ] [20808]

Исправлены ошибки автонумерации при настройке: цикл автонумерации - День, заполнять пропущенные за день:

•При длине автономера равной 2 символам и ведущих нулях неправильно создавались двузначные номера: 010 вместо 10, 011 вместо 11.

•Пропущенные номера искались не с единицы, а начиная с первого имеющегося номера. Так при наличии номеров 3,5,6 следующий автономер создавался равным не 1, а 4.

 

Разное

 

[ - ] [20767]

•Исправлен режим создания триггеров для контроля прикрепленных файлов [_авто_Триггер <Таблица> удаление файл]. Теперь  такие триггера создаются ТОЛЬКО для тех таблиц, которые имеют прикрепленные файлы. При привязке файла к таблице, еще не имеющей прикрепленных файлов, триггер для контроля удаления создается автоматически при добавлении записи в таблицу [Файл]. Для этого создан новый триггер [Триггер Файл Добавление] – входит в эталон текущей версии.

•Внесены изменения в скрипт, выполняющий чистку таблицы [Файл]  в рамках сервисного режима Создание/пересоздание триггеров для контроля прикрепленных файлов. Если таблица [Файл] входит в настройку репликации, ее чистка не выполняется.
